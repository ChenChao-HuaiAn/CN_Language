# CN_Language 项目技术规范和编码标准

## 文档概述

本文档定义了CN_Language项目的技术规范和编码标准。CN_Language 是一个中文编程语言，旨在为中文用户提供友好、直观的编程体验。本文档基于最新的架构设计原则，对项目的技术规范和编码标准进行了全面更新。

## 架构设计原则

CN_Language项目采用分层架构设计和SOLID设计原则，确保系统的可维护性、可扩展性和可测试性。详细架构设计请参考：
- [架构设计原则](../architecture/架构设计原则.md)
- [架构决策记录](../architecture/架构决策记录.md)
- [模块依赖规范](../architecture/模块依赖规范.md)
- [实施路线图](../architecture/实施路线图.md)

## 代码组织原则

### 单一职责原则
1. **每个.c文件不超过500行**：确保文件功能单一，易于理解和维护
2. **每个函数不超过50行**：函数功能明确，逻辑清晰
3. **每个头文件只暴露必要的接口**：隐藏实现细节，降低耦合度

### 分层架构原则
项目采用三层架构模型：
1. **基础设施层**：提供基础服务，无业务逻辑
2. **核心层**：实现编译器核心功能
3. **应用层**：提供用户界面和交互

### 模块化设计原则
1. **功能内聚性**：每个模块只负责一个功能领域
2. **接口隔离**：模块间通过抽象接口通信
3. **依赖管理**：明确声明和管理模块依赖关系




## 1. 项目命名规范

### 1.1 项目名称
- 正式项目名称：**CN_Language**（中文：CN语言）
- 命名含义：CN代表"Chinese"（中文），强调中文编程特性

### 1.2 文件命名规范

#### 1.2.1 项目源代码文件（C实现）
- **前缀要求**：所有核心C源代码文件必须使用`CN_`前缀;
              - 所有类名使用`CN_`前缀；
             
- **文件扩展名**：保持`.c`（源文件）和`.h`（头文件）扩展名
- **示例**：
  - `CN_main.c`（主程序文件）
  - `CN_language.c`（语言核心实现）
  - `CN_lexer.c`（词法分析器）
  - `CN_parser.c`（语法分析器）
  - `CN_ast.h`（AST头文件）

#### 1.2.2 CN语言源文件（用户编写的程序）
- **文件扩展名**：使用`.cn`作为CN语言源文件扩展名
- **示例**：
  - `hello_world.cn`（Hello World程序）
  - `test_basic.cn`（基础测试程序）
  - `examples/`目录下的所有示例程序

#### 1.2.3 可执行文件命名
- 主解释器：`cn_language.exe`（原`cc_language.exe`）
- 调试器：`cn_debugger.exe`（原`cc_debugger.exe`）
- REPL环境：`cn_repl.exe`（原`cc_repl.exe`）


## 2. 代码编写规范

### 2.1 命名约定

#### 2.1.1 结构体名
- 使用大驼峰命名法（PascalCase）
- 以`Stru_`前缀开头，以`_t`结尾表示类型，表示属于CN_Language项目
- 示例：
  ```c
  typedef struct Stru_LanguageCore_t Stru_LanguageCore_t;      // 语言核心结构体
  ```

#### 2.1.2 函数名
- 使用系统匈牙利命名法（Systems Hungarian）
- 公共函数使用描述性中文或英文名称
- 示例：
  ```c
  void F_ParseSourceCode();      // 解析源代码
  void F_ExecuteBytecode();      // 执行字节码
  void F_CompileToJIT();         // 编译为JIT代码
  ```

#### 2.1.3 变量名
- 使用小驼峰命名法
- 局部变量使用有意义的名称
- 全局变量以`g_`前缀开头
- 示例：
  ```c
  int cnLineNumber;             // 当前行号
  char* cnSourceCode;           // 源代码（C字符串）
  CN_ASTNode_t* g_astRoot;      // AST根节点（全局）
  ```

#### 2.1.4 常量名
- 使用全大写，下划线分隔
- 以`CN_`前缀开头
- 示例：
  ```c
  #define CN_MAX_TOKEN_LENGTH 256
  #define CN_VERSION "1.0.0"
  #define CN_PI 3.141592653589793
  ```

#### 2.1.5 枚举类型
- 枚举名使用大驼峰命名法
- 枚举值使用全大写，以枚举名开头
- 示例：
  ```c
  enum Eum_TokenType {
      Eum_TOKEN_KEYWORD,      // 关键字
      Eum_TOKEN_IDENTIFIER,   // 标识符
  };
  ```

### 2.2 代码格式规范

#### 2.2.1 缩进
- 使用4个空格进行缩进（禁止使用制表符）
- 示例：
  ```c
  void cnProcessToken() {
      if (isValidToken()) {
          parseToken();
          if (shouldContinue()) {
              processNextToken();
          }
      }
  }
  ```

#### 2.2.2 大括号风格
- 使用Allman风格（大括号单独成行）
- 示例：
  ```c
  void cnFunction()
  {
      if (condition)
      {
          // 代码块
      }
      else
      {
          // 其他代码块
      }
  }
  ```

#### 2.2.3 行长度
- 每行代码不超过120个字符
- 超长行应合理换行，保持可读性

#### 2.2.4 空格使用
- 运算符两侧添加空格
- 逗号后添加空格
- 控制语句关键字后添加空格
- 示例：
  ```c
  int result = a + b * c;
  for (int i = 0; i < count; i++)
  {
      processItem(items[i]);
  }
  ```

### 2.3 注释规范

#### 2.3.1 文件头注释
每个源文件开头应包含文件头注释：
```c
/******************************************************************************
 * 文件名: CN_language.c
 * 功能: CN_Language语言核心实现
 * 作者: CN_Language开发团队

 * 修改历史:

 *  2025-12-26: 添加字节码编译支持
 * 版权: MIT许可证
 ******************************************************************************/
```

#### 2.3.2 函数注释
每个函数前应添加注释说明：
```c
/**
 * @brief 解析CN语言源代码
 * @param sourceCode 源代码字符串（C字符串）
 * @param filename 源文件名（用于错误报告）
 * @return 解析成功返回true，失败返回false
 * @note 此函数执行词法分析和语法分析，生成AST
 */
bool F_ParseSourceCode(const char* sourceCode, const char* filename);
```

#### 2.3.3 行内注释
- 复杂逻辑处添加行内注释
- 注释使用中文，清晰说明代码意图
- 示例：
  ```c
  // 检查是否为中文标识符（Unicode范围）
  if (c >= 0x4E00 && c <= 0x9FFF)
  {
      return true;  // 是中文字符
  }
  ```

### 2.4 头文件规范

#### 2.4.1 头文件保护
每个头文件必须包含防止重复包含的保护：
```c
#ifndef CN_LANGUAGE_CORE_H
#define CN_LANGUAGE_CORE_H

// 头文件内容

#endif // CN_LANGUAGE_CORE_H
```

#### 2.4.2 包含顺序
头文件包含顺序应遵循以下规则：
1. 对应的源文件头文件（如有）
2. C标准库头文件
3. 第三方库头文件
4. 项目其他头文件

示例：
```c
#include "CN_language.h"      // 1. 对应源文件头文件

#include <stdio.h>              // 2. C标准库
#include <stdlib.h>

#include <some_third_party.h>   // 3. 第三方库

#include "CN_lexer.h"        // 4. 项目其他头文件
#include "CN_parser.h"
```
