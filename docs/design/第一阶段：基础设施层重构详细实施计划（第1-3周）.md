### 第2周详细规划：内存管理和工具库

#### 目标
建立统一的内存管理接口和实用工具函数库。

#### 详细任务分解

**任务2.1：统一内存管理接口（第1-2天）**
- 设计`CN_memory.h`接口：
  ```c
  // 内存分配器接口
  typedef struct Stru_CN_AllocatorInterface_t {
      void* (*allocate)(size_t size, const char* file, int line);
      void (*deallocate)(void* ptr, const char* file, int line);
      void* (*reallocate)(void* ptr, size_t new_size, const char* file, int line);
  } Stru_CN_AllocatorInterface_t;
  
  // 全局分配器设置和包装函数
  void* cn_malloc(size_t size);
  void cn_free(void* ptr);
  void* cn_realloc(void* ptr, size_t new_size);
  ```
- 实现多种分配器策略：
  - 系统分配器（包装malloc/free）
  - 调试分配器（边界检查、泄漏检测）
  - 对象池分配器（固定大小对象重用）
  - 区域分配器（批量分配，统一释放）

**任务2.2：内存调试工具（第3天）**
- 内存泄漏检测（分配/释放跟踪）
- 边界检查（前后保护字节）
- 使用统计（峰值内存、分配次数）
- 内存损坏检测（定期校验和）

**任务2.3：工具函数库（第4-5天）**
- 字符串处理：`CN_string.h`
  - UTF-8编码解码
  - 字符串分割、连接、格式化
  - 字符串查找和替换等等...
- 数学函数：`CN_math.h`
  - 安全算术（溢出检查）
  - 随机数生成器
  - 常用数学运算
- 错误处理框架：`CN_error.h`
  - 错误码定义（Eum_CN_ErrorCode_t）
  - 错误上下文传递
  - 错误链支持

**任务2.4：日志系统（第6天）**
- 设计`CN_log.h`接口：
  ```c
  typedef enum Eum_CN_LogLevel_t {
      Eum_LOG_LEVEL_DEBUG,
      Eum_LOG_LEVEL_INFO,
      Eum_LOG_LEVEL_WARNING,
      Eum_LOG_LEVEL_ERROR,
      Eum_LOG_LEVEL_FATAL

  void CN_log(Eum_CN_LogLevel_t level, const char* format, ...);
  ```

- 支持多输出目标（控制台、文件、网络）
- 日志级别过滤和格式化
- 线程安全和性能优化

**任务2.5：集成和测试（第7天）**
- 内存管理模块与containers模块集成测试
- 压力测试（长时间运行，大内存使用）
