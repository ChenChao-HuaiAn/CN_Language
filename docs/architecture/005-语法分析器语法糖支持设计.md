## 1. 概述

### 1.1 设计目标
本设计文档描述了CN_Language语法分析器的语法糖支持机制，旨在实现：
1. 提供更简洁、更易读的语法表达方式
2. 支持现代编程语言的常用语法糖
3. 保持语言核心的简洁性和一致性
4. 实现语法糖到标准语法的透明转换

### 1.2 设计原则
- **可读性优先**: 语法糖应提高代码可读性
- **一致性**: 语法糖转换应保持语义一致性
- **透明性**: 语法糖转换对用户透明
- **可扩展性**: 支持未来添加新的语法糖

## 2. 支持的语法糖类型

### 2.1 运算符语法糖
| 语法糖 | 标准语法 | 描述 |
|--------|----------|------|
| `a += b` | `a = a + b` | 复合赋值 |
| `a++` | `a = a + 1` | 后置自增 |
| `++a` | `a = a + 1` | 前置自增 |
| `a ?? b` | `a != 空 ? a : b` | 空值合并 |
| `a?.b` | `a != 空 ? a.b : 空` | 可选链 |

### 2.2 控制流语法糖
| 语法糖 | 标准语法 | 描述 |
|--------|----------|------|
| `for 元素 in 列表` | `for (i=0; i<长度(列表); i++) { 元素 = 列表[i]; ... }` | 范围循环 |
| `[表达式 for 元素 in 列表 if 条件]` | 列表推导式 | 列表推导 |
| `条件 ? 值1 : 值2` | `如果 (条件) { 值1 } 否则 { 值2 }` | 三元运算符 |

### 2.3 函数和表达式语法糖
| 语法糖 | 标准语法 | 描述 |
|--------|----------|------|
| `(参数) => 表达式` | `函数 (参数) { 返回 表达式 }` | 箭头函数 |
| `{ 属性 }` | `对象 { 属性: 属性 }` | 属性简写 |
| `函数名(参数) = 表达式` | `函数 函数名(参数) { 返回 表达式 }` | 单表达式函数 |

### 2.4 声明语法糖
| 语法糖 | 标准语法 | 描述 |
|--------|----------|------|
| `常量 名称 = 值` | `变量 名称 = 值` (不可变) | 常量声明 |
| `类型 名称` | `变量 名称: 类型` | 类型别名 |

## 4. 实现细节

### 4.1 核心数据结构

#### 4.1.1 语法糖规则结构
```c
typedef struct Stru_SyntaxSugarRule_t {
    char* name;                    // 规则名称
    Eum_SyntaxSugarType type;      // 语法糖类型
    Stru_PatternMatcher_t* pattern; // 匹配模式
    Stru_Transformer_t* transformer; // 转换器
    int priority;                  // 优先级 (0-100)
    bool enabled;                  // 是否启用
} Stru_SyntaxSugarRule_t;
```

#### 4.1.2 语法糖匹配上下文
```c
typedef struct Stru_SyntaxSugarContext_t {
    Stru_TokenStream_t* tokens;    // 令牌流
    Stru_ASTNode_t* current_node;  // 当前AST节点
    Stru_ParseState_t* parse_state; // 解析状态
    Stru_Scope_t* current_scope;   // 当前作用域
    Stru_TypeInfo_t* expected_type; // 期望类型
} Stru_SyntaxSugarContext_t;
```

### 4.2 关键算法

#### 4.2.1 语法糖检测算法
```c
Stru_SyntaxSugarMatch_t* F_detect_syntax_sugar(Stru_SyntaxSugarContext_t* context) {
    Stru_SyntaxSugarMatch_t* match = NULL;
    
    // 获取所有启用的语法糖规则
    Stru_SyntaxSugarRule_t** rules = F_get_enabled_syntax_sugar_rules();
    
    for (int i = 0; rules[i] != NULL; i++) {
        // 尝试匹配规则
        match = F_try_match_rule(rules[i], context);
        
        if (match != NULL) {
            // 找到匹配，返回
            return match;
        }
    }
    
    return NULL;
}
```

#### 4.2.2 语法糖转换算法
```c
Stru_ASTNode_t* F_transform_syntax_sugar(Stru_SyntaxSugarMatch_t* match) {
    // 获取转换器
    Stru_Transformer_t* transformer = match->rule->transformer;
    
    // 应用转换
    Stru_ASTNode_t* transformed = transformer->transform(
        transformer, 
        match->matched_tokens,
        match->context
    );
    
    // 验证转换结果
    if (!F_validate_transformation(transformed, match)) {
        // 转换验证失败，返回原始AST
        F_destroy_ast_node(transformed);
        return F_reconstruct_original_ast(match);
    }
    
    return transformed;
}
```

## 5. 具体语法糖实现

### 5.1 复合赋值语法糖

#### 5.1.1 转换规则
```
a += b  →  a = a + b
a -= b  →  a = a - b
a *= b  →  a = a * b
a /= b  →  a = a / b
```

#### 5.1.2 实现代码
```c
Stru_ASTNode_t* F_transform_compound_assignment(Stru_Transformer_t* transformer,
                                               Stru_Token_t** tokens,
                                               Stru_SyntaxSugarContext_t* context) {
    // tokens[0]: 左操作数标识符
    // tokens[1]: 复合赋值运算符 (+=, -=, *=, /=)
    // tokens[2]: 右操作数表达式
    
    // 提取运算符类型
    char* op = tokens[1]->value;
    char base_op = op[0];  // '+', '-', '*', '/'
    
    // 创建二元表达式: left op right
    Stru_ASTNode_t* binary_expr = F_create_binary_expression(
        F_create_identifier_node(tokens[0]->value),
        base_op,
        F_parse_expression(tokens[2], context->parse_state)
    );
    
    // 创建赋值表达式: left = binary_expr
    Stru_ASTNode_t* assignment = F_create_assignment_expression(
        F_create_identifier_node(tokens[0]->value),
        binary_expr
    );
    
    return assignment;
}
```

### 5.2 自增自减语法糖

#### 5.2.1 转换规则
```
a++  →  a = a + 1
++a  →  a = a + 1
a--  →  a = a - 1
--a  →  a = a - 1
```

#### 5.2.2 实现代码
```c
Stru_ASTNode_t* F_transform_increment_decrement(Stru_Transformer_t* transformer,
                                               Stru_Token_t** tokens,
                                               Stru_SyntaxSugarContext_t* context) {
    bool is_prefix = (tokens[0]->type == Eum_TOKEN_INCREMENT || 
                     tokens[0]->type == Eum_TOKEN_DECREMENT);
    bool is_increment = (tokens[0]->type == Eum_TOKEN_INCREMENT || 
                        (tokens[1] && tokens[1]->type == Eum_TOKEN_INCREMENT));
    
    char* identifier;
    if (is_prefix) {
        identifier = tokens[1]->value;
    } else {
        identifier = tokens[0]->value;
    }
    
    // 创建数值字面量 1
    Stru_ASTNode_t* one = F_create_numeric_literal(1);
    
    // 创建二元表达式
    char op = is_increment ? '+' : '-';
    Stru_ASTNode_t* binary_expr = F_create_binary_expression(
        F_create_identifier_node(identifier),
        op,
        one
    );
    
    // 创建赋值表达式
    Stru_ASTNode_t* assignment = F_create_assignment_expression(
        F_create_identifier_node(identifier),
        binary_expr
    );
    
    return assignment;
}
```

### 5.3 范围循环语法糖

#### 5.3.1 转换规则
```
for 元素 in 集合 {
    循环体
}
→
变量 __iter = 迭代器(集合)
当 __iter.有下一个() {
    变量 元素 = __iter.下一个()
    循环体
}
```

#### 5.3.2 实现代码
```c
Stru_ASTNode_t* F_transform_range_for_loop(Stru_Transformer_t* transformer,
                                          Stru_Token_t** tokens,
                                          Stru_SyntaxSugarContext_t* context) {
    // tokens[0]: "for"
    // tokens[1]: 元素变量名
    // tokens[2]: "in"
    // tokens[3]: 集合表达式
    // tokens[4...]: 循环体
    
    // 生成迭代器变量名
    char* iterator_var = F_generate_temp_variable("__iter");
    
    // 创建迭代器调用表达式
    Stru_ASTNode_t* iterator_call = F_create_function_call(
        F_create_identifier_node("迭代器"),
        F_parse_expression(tokens[3], context->parse_state)
    );
    
    // 创建迭代器变量声明
    Stru_ASTNode_t* iterator_decl = F_create_variable_declaration(
        iterator_var,
        iterator_call,
        NULL  // 无类型注解
    );
    
    // 创建循环条件
    Stru_ASTNode_t* has_next_call = F_create_method_call(
        F_create_identifier_node(iterator_var),
        "有下一个",
        NULL,  // 无参数
        0      // 参数数量
    );
    
    // 创建元素赋值
    Stru_ASTNode_t* next_call = F_create_method_call(
        F_create_identifier_node(iterator_var),
        "下一个",
        NULL,
        0
    );
    
    Stru_ASTNode_t* element_assignment = F_create_variable_declaration(
        tokens[1]->value,
        next_call,
        NULL
    );
    
    // 创建循环体（包含元素赋值和原始循环体）
    Stru_ASTNode_t* loop_body = F_create_block_statement();
    F_add_statement_to_block(loop_body, element_assignment);
    
    // 添加原始循环体语句
    Stru_ASTNode_t* original_body = F_parse_statements(tokens + 4, context->parse_state);
    F_merge_statements_into_block(loop_body, original_body);
    
    // 创建while循环
    Stru_ASTNode_t* while_loop = F_create_while_loop(has_next_call, loop_body);
    
    // 创建包含迭代器声明和while循环的块
    Stru_ASTNode_t* final_block = F_create_block_statement();
    F_add_statement_to_block(final_block, iterator_decl);
    F_add_statement_to_block(final_block, while_loop);
    
    return final_block;
}
```

## 6. 集成设计

### 6.1 与解析器集成
语法糖处理集成到解析流程的关键位置：

```c
// 在表达式解析中的集成
Stru_ASTNode_t* F_parse_expression_with_syntax_sugar(Stru_ParserInterface_t* parser) {
    // 1. 解析原始表达式
    Stru_ASTNode_t* original_expr = F_parse_expression(parser);
    
    // 2. 检测语法糖
    Stru_SyntaxSugarContext_t context = {
        .tokens = parser->token_stream,
        .current_node = original_expr,
        .parse_state = parser->internal_state,
        .current_scope = F_get_current_scope(parser),
        .expected_type = NULL
    };
    
    Stru_SyntaxSugarMatch_t* match = F_detect_syntax_sugar(&context);
    
    // 3. 如果检测到语法糖，进行转换
    if (match != NULL) {
        Stru_ASTNode_t* transformed = F_transform_syntax_sugar(match);
        F_destroy_syntax_sugar_match(match);
        
        // 验证转换后的表达式
        if (F_validate_expression(transformed, parser)) {
            return transformed;
        } else {
            F_destroy_ast_node(transformed);
            return original_expr;
        }
    }
    
    return original_expr;
}
```

### 6.2 与AST构建器集成
语法糖转换发生在AST构建阶段，确保：
1. 转换后的AST符合标准语法
2. 转换过程保持语义等价
3. 转换可逆（用于调试目的）

## 10. 错误处理

### 10.1 语法糖错误类型
1. **模式匹配错误**: 无法识别语法糖模式
2. **转换错误**: 语法糖转换失败
3. **验证错误**: 转换后的AST验证失败
4. **配置错误**: 语法糖配置无效

### 10.2 错误恢复策略
1. **回退到标准语法**: 转换失败时使用原始语法
2. **部分转换**: 只转换成功的部分
3. **用户提示**: 提供详细的错误信息和修复建议

## 11. 扩展性设计

### 11.1 插件式语法糖
支持通过插件添加新的语法糖：

```c
// 语法糖插件接口

    char* name;                    // 插件名称
    Stru_SyntaxSugarRule_t* rules; // 语法糖规则数组
    int rule_count;                // 规则数量
    bool (*initialize)(void);      // 初始化函数
    void (*cleanup)(void);         // 清理函数

```

### 11.2 动态语法糖

