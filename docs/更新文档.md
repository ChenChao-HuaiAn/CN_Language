### 🔄 核心数据结构

**异常对象结构体**：
```c
typedef struct CnException {
    const char *type_name;      // 异常类型名（UTF-8编码中文）
    size_t type_name_length;    // 类型名长度（字节数）
    const char *message;        // 异常消息（可选）
    size_t message_length;      // 消息长度
    const char *file;           // 抛出位置文件名
    int line;                   // 抛出位置行号
    void *user_data;            // 用户自定义数据（可选）
    bool is_rethrown;           // 是否为重新抛出
} CnException;
```

**异常帧结构体**：
```c
typedef struct CnExceptionFrame {
    jmp_buf jump_buffer;                // setjmp缓冲区
    struct CnExceptionFrame *prev;      // 前一帧（链表结构）
    CnException *current_exception;     // 当前捕获的异常
    int catch_handled;                  // catch块是否已处理
    int finally_executed;               // finally块是否已执行
} CnExceptionFrame;
```

### 🎯 支持的语法

**try-catch-finally 语法**：
```cn
尝试 {
    // 可能抛出异常的代码
} 捕获 (异常类型 e) {
    // 异常处理代码
} 最终 {
    // 无论是否异常都会执行的代码
}
```

**throw 语法**：
```cn
抛出 异常类型("异常消息");
抛出 e;  // 重新抛出
```

### 📝 技术实现要点

1. **setjmp/longjmp机制**：使用标准C的setjmp/longjmp实现异常跳转
2. **异常栈管理**：通过链表结构管理嵌套的try块
3. **类型匹配**：支持基于类型名的异常匹配
4. **代码生成集成**：在C代码生成阶段生成异常处理代码
5. **位置信息保留**：异常抛出时记录文件名和行号

### 🔧 异常处理流程

```
1. 进入try块：创建异常帧，压入异常栈
2. setjmp保存执行环境
3. 执行try块代码
4. 如果抛出异常：
   - 创建异常对象
   - longjmp跳转到异常帧
   - 查找匹配的catch块
5. 执行catch块（如果匹配）
6. 执行finally块
7. 弹出异常帧
```
