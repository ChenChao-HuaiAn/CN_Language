## 基本概念

### 模块
每个 `.cn` 文件都是一个**模块**。模块是代码组织的基本单位。

### 包
包含 `__包__.cn` 文件的目录被视为一个**包**。包可以包含多个模块和子包。

### 模块标识符
模块通过点分路径标识，例如：
- `数学` - 单层模块
- `工具.数学` - 嵌套在工具包中的数学模块
- `工具.数学.高级` - 多层嵌套

## 模块导入语法

### 1. 基本导入

```cn
导入 数学;
```

导入整个模块，通过 `数学.加法()` 访问其成员。

### 2. 选择性导入（推荐）

```cn
从 数学 导入 { 加法, 减法 };
```

只导入指定的成员，可以直接使用 `加法()` 而无需前缀。

### 3. 通配符导入

```cn
从 数学 导入 *;
```

导入模块的所有公开成员。**注意**：可能导致命名冲突。

### 4. 路径导入

```cn
从 工具/数学/高级 导入 { 平方根 };
```

导入嵌套包中的模块成员。

### 5. 相对导入

```cn
// 导入同目录下的兄弟模块
从 ./兄弟模块 导入 { 函数A };

// 导入父目录中的模块
从 ../父级模块 导入 { 函数B };

// 导入祖父目录中的模块
从 ../../祖父模块 导入 { 函数C };
```

相对导入使用路径前缀：
- `./` - 当前目录
- `../` - 父目录
- `../../` - 祖父目录

### 6. 导入别名

```cn
从 很长的模块名 导入 { 原名 作为 别名 };
```

为导入的符号指定别名，避免命名冲突。

## 包结构

### 标准项目结构

```
项目根目录/
├── 主程序.cn              # 入口文件
├── 工具/                   # 工具包
│   ├── __包__.cn          # 包初始化文件（必需）
│   ├── 数学/              # 数学子包
│   │   ├── __包__.cn      # 子包初始化文件
│   │   └── 数学.cn        # 数学模块
│   └── 文本/              # 文本子包
│       ├── __包__.cn
│       └── 处理.cn
└── 配置/
    └── 设置.cn
```

### 包初始化文件 `__包__.cn`

```cn
// __包__.cn - 使目录成为包

// 重新导出子模块（可选）
公开:
从 ./数学 导入 *;
从 ./文本 导入 *;

// 包级别的变量和函数
整数 包版本 = 1;
```

## 可见性控制

### 公开声明

```cn
公开:

函数 可以被导入的函数() -> 整数 {
    返回 1;
}

整数 公开变量 = 100;
```

`公开:` 后的所有声明可以被其他模块导入。

### 私有声明

```cn
私有:

函数 内部函数() -> 整数 {
    返回 42;
}

整数 内部变量 = 0;
```

`私有:` 后的声明只能在当前模块内使用。

### 默认可见性

未包含在 `公开:` 或 `私有:` 之后的声明默认为**模块内可见**。

## 命令行参数

### 基本用法

```bash
# 编译单个文件
cnc 主程序.cn -o 程序

# 编译多个文件
cnc 模块A.cn 模块B.cn 主程序.cn -o 程序
```

### 模块搜索路径 `-I`

```bash
# 添加模块搜索路径
cnc 主程序.cn -I./工具 -I./外部库 -o 程序
```

编译器会在指定路径中查找导入的模块。

### 指定入口文件 `--main`

```bash
# 显式指定入口文件
cnc 模块A.cn 模块B.cn --main 主程序.cn -o 程序
```

### 项目编译 `--project`

```bash
# 编译整个项目目录
cnc --project ./src -o 程序
```

自动发现并编译目录中的所有 `.cn` 文件。

### 环境变量

| 变量名 | 说明 |
|--------|------|
| `CN_MODULE_PATH` | 模块搜索路径（多个路径用分号分隔） |
| `CN_RUNTIME_PATH` | 运行时库路径 |
| `CN_RUNTIME_HEADER_PATH` | 运行时头文件路径 |

## 最佳实践

### 1. 优先使用选择性导入

```cn
// 推荐
从 数学 导入 { 加法, 减法 };

// 不推荐
从 数学 导入 *;
```

选择性导入使依赖关系更清晰，减少命名冲突。

### 2. 避免循环导入

如果模块 A 导入模块 B，模块 B 又导入模块 A，会形成循环依赖。
编译器会检测并报错。

**解决方案**：
- 重构代码，提取共享部分到第三个模块
- 使用依赖注入模式

### 3. 合理使用可见性

- 将实现细节放在 `私有` 块中
- 只公开必要的接口
- 在 `__包__.cn` 中重新导出子模块的公开接口

### 4. 模块命名规范

- 使用有意义的中文名称
- 避免使用过长的模块名
- 包名使用名词，模块名反映其功能

## 错误处理

### 常见错误

1. **模块未找到**
   ```
   错误: 无法找到模块 "不存在的模块"
   ```
   检查模块路径是否正确，或使用 `-I` 添加搜索路径。

2. **循环导入**
   ```
   错误: 检测到循环导入: A -> B -> A
   ```
   重构代码消除循环依赖。

3. **私有成员访问**
   ```
   错误: 无法访问私有成员 "内部函数"
   ```
   私有成员不能从其他模块导入。

4. **名称冲突**
   ```
   错误: 符号 "加法" 已存在
   ```
   使用别名或选择性导入解决冲突。
