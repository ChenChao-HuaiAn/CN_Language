# CN_pool_operations 模块

## 概述

`CN_pool_operations.c` 是CN_Language对象池分配器的操作模块，负责对象的分配、释放和重新分配等核心操作。

## 功能职责

### 核心操作函数
1. **对象分配**
   - `CN_pool_alloc()`: 从对象池分配一个对象
   - 支持空闲链表重用
   - 支持自动扩展

2. **对象释放**
   - `CN_pool_free()`: 释放对象到对象池
   - 重复释放检测
   - 无效指针检测

3. **对象重新分配**
   - `CN_pool_realloc()`: 重新分配对象（兼容接口）
   - 支持NULL指针分配
   - 数据复制功能

### 辅助功能
- 空闲链表管理
- 内存清零控制
- 错误处理

## 设计原则

### 单一职责
本模块专注于对象的分配、释放和重新分配操作，不涉及池的创建、销毁或统计功能。

### 性能优化
1. **空闲链表**: 快速重用最近释放的对象
2. **位图管理**: 高效跟踪对象使用状态
3. **缓存友好**: 连续内存布局提高缓存命中率

### 安全特性
1. **边界检查**: 验证指针有效性
2. **重复释放检测**: 防止同一指针多次释放
3. **内存清零**: 可选的分配/释放时清零

## 算法实现

### 分配算法
```c
1. 检查空闲链表
   - 如果有空闲对象，直接返回
   
2. 遍历内存块
   - 查找第一个有空闲位置的块
   - 使用位图快速定位空闲位置
   
3. 自动扩展
   - 如果所有块都满且允许扩展，创建新块
   - 递归调用分配函数
```

### 释放算法
```c
1. 参数验证
   - 检查指针是否为NULL
   - 检查池是否有效
   
2. 重复释放检测
   - 遍历空闲链表，检查是否已释放
   
3. 查找所属块
   - 遍历所有块，查找指针所属块
   - 验证指针对齐和范围
   
4. 更新状态
   - 更新位图标记
   - 添加到空闲链表
   - 更新统计信息
```

## 使用示例

```c
#include "CN_pool_allocator.h"

// 分配对象
void* obj1 = CN_pool_alloc(pool);
if (obj1 == NULL) {
    // 处理分配失败
}

// 使用对象
MyData* data = (MyData*)obj1;
data->value = 42;

// 释放对象
if (!CN_pool_free(pool, obj1)) {
    // 处理释放失败
}

// 重新分配
void* new_obj = CN_pool_realloc(pool, obj2, new_size);
```

## 性能特性

### 时间复杂度
- 分配（空闲链表）: O(1)
- 分配（位图查找）: O(n) - n为块数量
- 释放: O(n) - n为块数量
- 重新分配: O(1) - 如果大小相同

### 空间开销
- 空闲链表: 使用嵌入链表，无额外开销
- 位图: 每个对象1位开销
