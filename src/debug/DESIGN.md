# CN_debug 设计文档

## 设计目标

CN_debug调试系统的设计目标是为CN_Language项目提供一个高效、灵活且零开销的调试基础设施。主要目标包括：

1. **模块化设计** - 遵循单一职责原则，专注于调试功能
2. **性能优化** - 生产环境零开销，开发环境高效调试
3. **易用性** - 简洁的API接口，直观的使用方式
4. **可扩展性** - 支持多种输出方式和级别控制
5. **跨平台兼容** - 支持Windows和Linux平台
6. **中文友好** - 完美支持UTF-8编码的中文字符

## 架构设计

### 整体架构

```
┌─────────────────┐
│   应用程序代码   │
└─────────┬───────┘
          │ #include "CN_debug.h"
          ▼
┌─────────────────┐
│   CN_DEBUG宏    │
│   CN_ASSERT宏   │
└─────────┬───────┘
          │ 条件编译
          ▼
┌─────────────────┐    ┌─────────────────┐
│  调试版本代码   │    │  发布版本代码   │
│ (功能完整)      │    │ (零开销)        │
└─────────┬───────┘    └─────────────────┘
          │
          ▼
┌─────────────────┐
│  F_debug_log    │
│ F_debug_assert  │
│  F_debug_init   │
│    等函数        │
└─────────────────┘
          │
          ▼
┌─────────────────┐
│   输出系统      │
│ (stdout/stderr) │
│   (文件输出)    │
└─────────────────┘
```

### 核心组件

#### 1. 条件编译系统
通过`CN_DEBUG_MODE`宏实现条件编译：
- 定义时：调试代码被编译进程序
- 未定义时：调试代码被编译器优化掉

#### 2. 分级调试系统
四级调试级别提供精细的控制：
- `NONE` - 不输出任何调试信息
- `ERROR` - 只输出错误信息
- `WARN` - 输出错误和警告信息
- `INFO` - 输出错误、警告和一般信息
- `DEBUG` - 输出所有级别的调试信息

#### 3. 输出管理系统
支持多种输出方式：
- 标准错误输出（默认）
- 文件输出（可配置）
- 自动资源管理

## 设计原则

### 1. 单一职责原则
每个函数都有明确的职责：
- `F_debug_init` - 初始化调试系统
- `F_debug_log` - 记录调试信息
- `F_debug_assert` - 执行断言检查
- `F_debug_set_output_file` - 管理输出目标

### 2. 零开销原则
通过条件编译实现生产环境零开销：
```c
#ifdef CN_DEBUG_MODE
    #define CN_DEBUG(level, format, ...) \
        F_debug_log(level, __FILE__, __LINE__, format, ##__VA_ARGS__)
#else
    #define CN_DEBUG(level, format, ...)
#endif
```

### 3. 资源管理原则
自动管理文件资源，防止内存泄漏：
- 初始化时清理旧资源
- 设置新输出时关闭旧文件
- 程序结束时自动清理

### 4. 线程安全原则
基础实现保证基本的线程安全性：
- 全局状态使用静态变量
- 文件I/O操作具有基本原子性
- 为复杂多线程场景提供扩展空间

## 技术实现

### 1. 宏设计
使用可变参数宏实现灵活的调试输出：
```c
#define CN_DEBUG(level, format, ...) \
    F_debug_log(level, __FILE__, __LINE__, format, ##__VA_ARGS__)
```

### 2. 时间戳处理
使用标准C库的时间函数生成时间戳：
```c
time_t current_time;
time(&current_time);
struct tm* time_info = localtime(&current_time);
```

### 3. 文件名处理
智能处理文件路径，只显示文件名：
```c
const char* filename = strrchr(file, '/');
if (filename == NULL)
{
    filename = strrchr(file, '\\');
}
if (filename == NULL)
{
    filename = file;
}
else
{
    filename++; // 跳过路径分隔符
}
```

### 4. 内存管理
使用标准库函数管理内存：
- `strdup` - 复制字符串
- `free` - 释放内存
- 自动资源清理

## 性能优化

### 1. 编译时优化
- 未定义`CN_DEBUG_MODE`时，调试代码完全消失
- 减少了函数调用开销
- 减少了条件判断开销

### 2. 运行时优化
- 级别检查快速返回
- 缓冲I/O提高输出效率
- 避免不必要的字符串处理

### 3. 内存优化
- 最小化全局变量使用
- 及时释放分配的内存
- 避免内存碎片

## 最佳实践

### 1. 调试级别使用建议
- `ERROR` - 系统错误、资源不足、不可恢复错误
- `WARN` - 可恢复错误、性能问题、潜在问题
- `INFO` - 重要事件、状态变更、用户操作
- `DEBUG` - 详细调试信息、变量值、流程跟踪

### 2. 性能监控
- 生产环境建议使用`ERROR`或`WARN`级别
- 高频调试场景注意性能影响
- 定期清理日志文件

### 3. 安全考虑
- 避免在调试信息中输出敏感数据
- 文件输出注意权限设置
- 网络环境谨慎使用文件输出
