# CN_Language 数据类型模块 - 设计文档

## 概述

本文档详细说明CN_Language数据类型模块的设计原理、架构决策和技术实现细节。数据类型模块是CN_Language编译器和运行时的核心组件，负责定义、管理和操作所有数据类型。

## 设计目标

### 1. 核心设计原则

1. **类型安全**：提供编译时和运行时的类型检查机制
2. **内存安全**：实现引用计数和自动内存管理
3. **可扩展性**：支持用户自定义类型和类型系统扩展
4. **性能优化**：高效的类型操作和内存管理
5. **跨平台兼容**：支持Windows、Linux和macOS系统

### 2. 设计约束

- 每个.c文件不超过500行（单一职责原则）
- 每个函数不超过50行（功能单一性）
- 头文件只暴露必要的接口（信息隐藏）
- 支持UTF-8编码和中文标识符
- 遵循项目命名规范和编码标准

## 架构设计

### 1. 类型系统层次结构

```
类型系统 (CN_Type_t)
├── 基本类型 (Basic Types)
│   ├── 整数 (CN_TYPE_INTEGER)
│   ├── 小数 (CN_TYPE_DECIMAL)
│   ├── 字符串 (CN_TYPE_STRING)
│   └── 布尔 (CN_TYPE_BOOLEAN)
├── 复合类型 (Composite Types)
│   ├── 数组 (CN_TYPE_ARRAY)
│   ├── 结构体 (CN_TYPE_STRUCT)
│   └── 枚举 (CN_TYPE_ENUM)
├── 指针类型 (CN_TYPE_POINTER)
└── 引用类型 (CN_TYPE_REFERENCE)
```

#### 2.2 字符串结构体 (Stru_CN_String_t)
```c
typedef struct Stru_CN_String_t
{
    char* data;           // 字符串数据（UTF-8编码）
    size_t length;        // 字符串长度（字节数）
    int ref_count;        // 引用计数

```

#### 2.3 数组结构体 (Stru_CN_Array_t)
```c
typedef struct Stru_CN_Array_t

    size_t length;           // 数组长度
    size_t capacity;         // 数组容量
    int ref_count;           // 引用计数

```

### 3. 内存管理设计

#### 3.1 引用计数机制

```c
// 引用计数管理策略
typedef struct
{
    int ref_count;          // 当前引用计数
    void (*destroy)(void*); // 销毁函数指针
} RefCounted_t;

// 引用计数规则：
// 1. 创建对象时 ref_count = 1
// 2. 复制引用时 ref_count++
// 3. 释放引用时 ref_count--
// 4. 当 ref_count == 0 时调用销毁函数
```

#### 3.2 内存分配策略

1. **小对象池**：频繁创建的小对象使用对象池
2. **大对象直接分配**：大对象直接使用malloc/free
3. **对齐分配**：确保内存对齐以提高访问效率
4. **安全分配**：所有分配都进行NULL检查

### 4. 类型转换设计

#### 4.1 转换规则矩阵

| 源类型 → 目标类型 | 整数 | 小数 | 字符串 | 布尔 |
|------------------|------|------|--------|------|
| **整数**         | 直接 | 转换 | 格式化 | ≠0→真 |
| **小数**         | 截断 | 直接 | 格式化 | ≠0→真 |
| **字符串**       | 解析 | 解析 | 直接   | 非空→真 |
| **布尔**         | 真→1 | 真→1.0 | 格式化 | 直接 |

#### 4.2 转换函数设计

```c

// 转换函数注册表
```

## 设计决策记录

### 1. 为什么选择引用计数而不是垃圾回收？

**决策**：选择引用计数作为主要内存管理机制。

**理由**：
1. **确定性释放**：引用计数提供确定性的内存释放
2. **低延迟**：没有垃圾回收的暂停时间
3. **简单实现**：引用计数实现相对简单
4. **渐进式**：可以逐步扩展到完整的垃圾回收

**权衡**：
- 循环引用需要额外处理
- 引用计数操作有性能开销

### 2. 为什么使用UTF-8编码？

**决策**：使用UTF-8作为字符串编码。

**理由**：
1. **兼容性**：UTF-8与ASCII兼容
2. **空间效率**：英文字符使用1字节，中文字符使用3-4字节
3. **广泛支持**：现代操作系统和工具都支持UTF-8
4. **网络友好**：UTF-8是网络传输的标准编码

### 3. 为什么选择C语言实现？

**决策**：使用C语言实现核心数据类型模块。

**理由**：
1. **性能**：C语言提供接近硬件的性能
2. **可移植性**：C语言在所有平台上都有良好支持
3. **简单性**：C语言语法简单，易于理解和维护
4. **互操作性**：C语言可以轻松与其他语言互操作

**权衡**：
- 需要手动内存管理
- 缺少现代语言的高级特性
