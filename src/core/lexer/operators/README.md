# 运算符识别模块

## 概述

运算符识别模块负责识别CN_Language的各种运算符，包括算术运算符、比较运算符、赋值运算符等。它提供高效的运算符查找和识别功能，支持将运算符词素映射到相应的令牌类型。

## 文件说明

### CN_lexer_operators.h
- 运算符识别函数声明
- 运算符表定义
- 公共接口函数

### CN_lexer_operators.c
- 运算符识别实现
- 运算符表初始化
- 查找算法实现

## 功能特性

### 1. 运算符支持
- **算术运算符**：+、-、*、/、%
- **比较运算符**：==、!=、<、>、<=、>=
- **赋值运算符**：=、+=、-=、*=、/=
- **逻辑运算符**：&&、||、!
- **位运算符**：&、|、^、~、<<、>>

### 2. 运算符类别
- **单字符运算符**：+、-、*、/等
- **双字符运算符**：==、!=、<=、>=等
- **组合运算符**：+=、-=、*=、/=等
- **特殊运算符**：++、--等

### 3. 识别算法
- **长度优先** - 首先尝试匹配多字符运算符
- **精确匹配** - 确保准确的运算符识别
- **错误恢复** - 在无效运算符情况下的恢复机制

## 主要函数

### 识别函数
- `F_identify_operator()` - 识别运算符类型
- `F_is_operator_char()` - 检查字符是否为运算符字符
- `F_is_operator()` - 检查词素是否为有效运算符

### 辅助函数
- `F_initialize_operator_table()` - 初始化运算符表
- `F_destroy_operator_table()` - 清理运算符表

## 使用示例

```c
#include "CN_lexer_operators.h"

// 识别运算符
const char* lexeme = "+";
Eum_TokenType type = F_identify_operator(lexeme);

if (type != Eum_TOKEN_ERROR) {
    printf("识别到运算符: %s -> %d\n", lexeme, type);
}

// 检查字符是否为运算符字符
char c = '+';
bool is_op_char = F_is_operator_char(c);
if (is_op_char) {
    printf("%c 是运算符字符\n", c);
}
```

## 运算符表设计

### 数据结构
- **静态数组** - 编译时初始化的运算符表
- **按长度分组** - 单字符和双字符运算符分开存储
- **快速查找** - 基于长度的快速定位

### 查找算法
1. **长度检查** - 首先检查运算符长度
2. **分组查找** - 在相应长度组中查找
3. **精确匹配** - 验证字符串完全匹配
4. **类型返回** - 返回对应的令牌类型

## 维护指南

### 修改运算符表
1. 更新运算符表定义
2. 重新初始化查找表
