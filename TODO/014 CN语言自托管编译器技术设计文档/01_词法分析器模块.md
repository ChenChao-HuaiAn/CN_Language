## 1. 任务概述

### 1.1 任务目标

将现有C语言实现的词法分析器（`src/frontend/lexer/`）使用CN语言重写，实现自托管编译器的词法分析功能。

### 1.2 功能要求

| 功能项 | 说明 | 优先级 |
|--------|------|--------|
| 词元扫描 | 识别标识符、关键字、数字、字符串、运算符 | ⭐⭐⭐ 高 |
| 关键字识别 | 识别CN语言所有关键字（函数、变量、如果等） | ⭐⭐⭐ 高 |
| 错误恢复 | 遇到非法字符时的错误恢复机制 | ⭐⭐ 中 |
| 源位置追踪 | 记录每个词元的行号、列号 | ⭐⭐⭐ 高 |
| 注释处理 | 支持单行注释和多行注释 | ⭐⭐ 中 |

### 1.3 验收标准

- [ ] 词法分析器能正确扫描所有CN语言词元
- [ ] 关键字表完整覆盖所有CN语言关键字
- [ ] 错误恢复机制正常工作
- [ ] 词法分析测试用例100%通过
- [ ] 与C版本词法分析器输出一致

### 3.2 扫描器类设计

```cn
// src_cn/词法分析器/扫描器.cn
模块 词法分析器 {
公开:
    类 扫描器 {
    私有:
        字符串 源码;
        整数 位置;
        整数 行号;
        整数 列号;
        诊断管理器* 诊断;
        
    公开:
        函数 创建(字符串 源码, 诊断管理器* 诊断) -> 扫描器*;
        函数 销毁(扫描器* 实例) -> 空类型;
        函数 下一个词元() -> 词元;
        函数 预览词元() -> 词元;
        
    私有:
        函数 跳过空白字符() -> 空类型;
        函数 跳过注释() -> 布尔;
        函数 扫描标识符() -> 词元;
        函数 扫描数字() -> 词元;
        函数 扫描字符串() -> 词元;
        函数 扫描字符() -> 词元;
        函数 扫描运算符() -> 词元;
    }
}
```
