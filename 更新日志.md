## 功能完善总结

### 修复代码生成器指针/值类型混淆问题

**问题描述**：
当编译src_cn项目时，生成的C代码错误地使用 `.` 而不是 `->` 访问指针类型的成员。例如：`r55.名称` 应该是 `r55->名称`，因为 `cn_var_函数声明_7` 是 `struct 函数声明*` (指针类型)。

**根本原因**：
1. IR生成器中数组索引访问 (`CN_AST_EXPR_INDEX`) 调用 `cn_rt_array_get_element` 函数，该函数返回 `void*` 指针类型，但IR生成器将返回类型设置为 `expr->type`（元素类型），而不是指向元素类型的指针
2. 当 `expr->type` 为 NULL 时，返回类型也是 NULL，导致代码生成器无法正确识别指针类型

**修复内容**：

1. **IR生成器修复** ([`src/ir/gen/irgen.c:1180-1193`](src/ir/gen/irgen.c:1180))：
   - 修复数组索引访问的返回类型：`cn_rt_array_get_element` 返回 `void*`，所以返回类型应该是指向元素类型的指针
   - 当 `expr->type` 为 NULL 时，默认使用 `void*` 指针类型

2. **代码生成器修复** ([`src/backend/cgen/cgen.c:1593-1603`](src/backend/cgen/cgen.c:1593))：
   - 添加对CALL指令的指针类型强制更新逻辑
   - 确保函数调用的指针返回类型正确传播到寄存器类型

**验证结果**：
- 修复前：`r8 = *(void*)cn_rt_array_get_element(...); r9 = r8.名称;` (错误)
- 修复后：`r8 = *(void**)cn_rt_array_get_element(...); r9 = r8->名称;` (正确)

**修改文件**：
- `src/ir/gen/irgen.c` - 修复数组索引访问的返回类型
- `src/backend/cgen/cgen.c` - 添加CALL指令的指针类型传播处理
