## 功能完善总结

### 修复内容

#### 1. void返回类型推断修复
**文件**: `src/ir/gen/irgen.c`

**问题**: 当子类调用基类的void方法时，表达式类型被错误设置为非void类型，导致生成的C代码尝试对void函数结果赋值。

**修复**: 在IR生成阶段，当检测到调用的是基类void方法时，正确设置表达式类型为void。

```c
// 修复位置: irgen.c 第820行附近
// 检查基类方法的返回类型，如果是void则设置expr->type为void类型
```

#### 2. 基类构造函数调用修复
**文件**: `src/backend/cgen/class_cgen.c`

**问题**: 当基类没有显式构造函数时，仍然生成了对基类构造函数的调用，导致链接错误。

**修复**: 只有当基类有显式构造函数时才生成调用代码。

```c
// 修复位置: class_cgen.c
// 检查基类是否有构造函数，避免调用不存在的函数
```
