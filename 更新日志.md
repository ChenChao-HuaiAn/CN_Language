#### 修改内容

**采用两遍扫描方案解决结构体依赖问题**：

1. **第一遍扫描**：输出所有结构体的前向声明
   ```c
   // CN Language Global Struct Forward Declarations
   struct 结构体A;
   struct 结构体B;
   ```

2. **第二遍扫描**：输出所有结构体的完整定义
   ```c
   // CN Language Global Struct Definitions
   struct 结构体A {
       long long 值;
       struct 结构体B* 指向B;
   };
   
   struct 结构体B {
       long long 数据;
       struct 结构体A* 指向A;
   };
   ```

#### 解决的问题

当结构体A包含结构体B类型的成员，但结构体B的定义在结构体A之后时，C编译器会报"结构体未定义"错误。通过先输出前向声明，再输出完整定义，解决了这个问题。
