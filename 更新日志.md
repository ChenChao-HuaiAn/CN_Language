### 指针成员访问代码生成修复

**修复时间**: 2026-04-09 04:37

**问题描述**:
当指针变量被加载到寄存器后，MEMBER_ACCESS指令错误地使用了 `.` 运算符而不是 `->` 运算符，导致C编译错误。

**示例错误**:
```c
struct 函数声明* cn_var_函数声明_7;  // 指针类型变量
r55 = cn_var_函数声明_7;
r56 = r55.名称;  // 错误：应该是 r55->名称
```

**修复内容**:
修改 [`src/backend/cgen/cgen.c`](src/backend/cgen/cgen.c:1387) 中LOAD指令的类型传播逻辑：
- 优先从 `src1.type` 获取类型（这是最可靠的类型来源）
- 这确保指针类型信息正确传播到 `reg_types` 数组
- MEMBER_ACCESS指令现在能正确识别指针类型并使用 `->` 运算符

**验证结果**:
```c
r0 = cn_var_声明指针_0;
r1 = r0->名称;  // ✅ 正确使用了 -> 而不是 .
```
