### 变量类型推断和结构体成员访问类型推断修复

**修复时间**: 2026-04-10

### 问题分析

`arg_kind=20` 错误对应的是 `CN_TYPE_UNKNOWN`（未知类型），说明类型推断系统在某些情况下返回了未知类型。具体问题：

1. **变量声明时类型推断失败** - 当变量声明没有显式类型（如 `变量 保护宏 = 创建输出缓冲区(256);`），`变量节点.类型` 为 `无`，符号的 `类型信息` 被设置为 `无`
2. **语义检查阶段没有更新符号类型** - `检查变量声明` 函数检查了初始化表达式的类型，但没有将推断出的类型更新到符号表中
3. **成员访问类型推断依赖变量类型** - 如果变量符号的类型信息是 `无`，成员访问就会失败

### 修复内容

修改 [`src_cn/语义分析器/语义检查.cn`](src_cn/语义分析器/语义检查.cn:230) 中的 `检查变量声明` 函数：

1. **重构类型推断逻辑** - 将初始化表达式的类型检查提前，并保存推断的类型
2. **添加类型推断更新** - 当变量没有显式类型时，从初始化表达式推断类型并更新到变量节点和符号表
3. **更新符号表类型信息** - 通过 `查找符号` 找到变量符号，并更新其 `类型信息` 字段

### 关键代码修改

```cn
// 检查初始化表达式并推断类型
变量 推断类型 = 无;
如果 (变量节点.初始值 != 无) {
    推断类型 = 检查表达式(上下文, 变量节点.初始值);
}

// 检查显式类型
如果 (变量节点.类型 != 无) {
    // ... 类型兼容性检查
} 否则 {
    // 没有显式类型，从初始化表达式推断类型
    如果 (推断类型 != 无) {
        // 将推断的类型设置到变量节点
        变量节点.类型 = 类型信息转类型节点(推断类型);
        
        // 更新符号表中的类型信息
        变量 符号指针 = 查找符号(上下文.符号表, 变量节点.名称);
        如果 (符号指针 != 无) {
            符号指针.类型信息 = 变量节点.类型;
        }
    }
}
```
