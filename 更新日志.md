### 2026-04-07 函数调用返回值处理验证

#### 任务背景
用户报告编译器在处理 `如果 (函数调用())` 这样的代码时，没有正确获取函数返回值，导致生成的C代码中出现 `if (/* NONE */)` 这样的无效代码。

#### 调查结果
经过详细分析和测试验证，发现编译器**已经正确处理**函数调用返回值：

1. **IR生成器** ([`src/ir/gen/irgen.c:618-1017`](src/ir/gen/irgen.c:618))
   - `CN_AST_EXPR_CALL` 分支正确处理函数调用
   - 分配结果寄存器：`call_inst->dest = cn_ir_op_reg(dest_reg, expr->type)`
   - 返回调用指令的目标操作数：`return call_inst->dest`

2. **代码生成器** ([`src/backend/cgen/cgen.c:724`](src/backend/cgen/cgen.c:724))
   - `CN_IR_INST_BRANCH` 指令正确生成：`if (inst->src1) goto ...`

#### 测试验证
创建测试用例验证多种场景：

| 测试场景 | 测试文件 | 结果 |
|---------|---------|------|
| 函数调用作为if条件 | `tests/函数调用返回值测试.cn` | ✅ 正确 |
| 函数调用作为while条件 | `tests/函数调用返回值测试2.cn` | ✅ 正确 |
| 函数调用在表达式中 | `tests/函数调用返回值测试2.cn` | ✅ 正确 |
| 嵌套函数调用 | `tests/函数调用返回值测试2.cn` | ✅ 正确 |

#### 生成的C代码示例
```c
// CN语言: 如果 (是正数(数值)) { ... }
r1 = 是正数(r0);
if (r1) goto if_then_0; else goto if_else_1;
```

#### 结论
编译器已正确处理函数调用返回值，无需修复。问题可能已在之前的开发中解决，或问题描述的场景不存在。
