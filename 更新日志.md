## 功能完善总结

### 修复IR生成器类型大小函数处理Bug

**问题描述**：
IR生成器在处理 `类型大小(IR生成上下文)` 函数调用时，错误地将类型名 `IR生成上下文` 处理为变量名 `cn_var_IR生成上下文`。

**修复内容**：

1. **添加IR指令** - 在 [`include/cnlang/ir/ir.h`](include/cnlang/ir/ir.h:59) 中添加 `CN_IR_INST_SIZEOF` 指令

2. **修改IR生成器** - 在 [`src/ir/gen/irgen.c`](src/ir/gen/irgen.c:707) 中识别 `类型大小` 函数调用：
   - 检查参数是否是类型名（结构体、类、枚举）
   - 生成 `CN_IR_INST_SIZEOF` 指令而不是函数调用

3. **修改代码生成器** - 在 [`src/backend/cgen/cgen.c`](src/backend/cgen/cgen.c:733) 中处理 `CN_IR_INST_SIZEOF` 指令：
   - 当参数是类型名时，生成 `sizeof(struct 类型名)` 或 `sizeof(enum 类型名)`
   - 当参数是变量时，生成 `sizeof(变量类型)`

**验证结果**：

CN源代码：
```cn
结构体 IR生成上下文 {
    整数 数据;
};

函数 主程序() -> 整数 {
    IR生成上下文 IR上下文;
    整数 大小 = 类型大小(IR生成上下文);
    返回 大小;
}
```

生成的C代码：
```c
struct IR生成上下文 {
    long long 数据;
};

long long main() {
  struct IR生成上下文 cn_var_IR上下文_0;
  long long cn_var_大小_1;
  r0 = sizeof(struct IR生成上下文);  // 正确！
  cn_var_大小_1 = r0;
  return r1;
}
```

**编译器构建**：cnc.exe 已成功重新构建
