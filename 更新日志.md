### 修复内容

#### 1. 词法分析器UTF-8 BOM处理

**问题**: 编译器无法正确处理带有UTF-8 BOM的源文件，导致所有src_cn文件编译失败。

**错误现象**:
```
[DEBUG PARSER] 无法识别的token: kind=1, line=1, col=1
解析失败
```

**根本原因**: 词法分析器在初始化时没有跳过UTF-8 BOM（0xEF 0xBB 0xBF），导致BOM被当作标识符处理。

**修复方案**: 在 [`src/frontend/lexer/lexer.c:191`](src/frontend/lexer/lexer.c:191) 的 `cn_frontend_lexer_init` 函数中添加BOM检测和跳过逻辑：

```c
// 跳过UTF-8 BOM（如果存在）
// UTF-8 BOM: 0xEF 0xBB 0xBF
if (length >= 3 && 
    (unsigned char)source[0] == 0xEF &&
    (unsigned char)source[1] == 0xBB &&
    (unsigned char)source[2] == 0xBF) {
    lexer->offset = 3;  // 跳过BOM的3个字节
}
```
