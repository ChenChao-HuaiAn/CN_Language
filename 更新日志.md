## 功能完善总结

### 代码生成器寄存器声明遗漏问题修复

**问题描述**：
代码生成器在生成C代码时，某些寄存器（如 `r1`、`r10`）被使用但未声明，导致C编译错误 `error C2065: "r1": 未声明的标识符`。

**根本原因**：
代码生成器的寄存器类型收集逻辑在声明寄存器时，没有处理 `CN_TYPE_VOID` 类型的寄存器。当函数调用的返回值类型被错误地设置为 `void` 时，对应的寄存器就不会被声明。

**修复内容**：
在 [`src/backend/cgen/cgen.c:1601`](src/backend/cgen/cgen.c:1601) 中，将 `CN_TYPE_VOID` 类型添加到整型寄存器声明条件中：

```c
// 修复前
if (!reg_types[i] || reg_types[i]->kind == CN_TYPE_INT || reg_types[i]->kind == CN_TYPE_UNKNOWN)

// 修复后
if (!reg_types[i] || reg_types[i]->kind == CN_TYPE_VOID || reg_types[i]->kind == CN_TYPE_INT || reg_types[i]->kind == CN_TYPE_UNKNOWN)
```

**验证结果**：
- 创建测试文件 `tests/寄存器声明测试.cn` 和 `tests/寄存器声明测试2.cn`
- 编译成功，生成的C代码中所有使用的寄存器都被正确声明
