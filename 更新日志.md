## 功能完善总结

### P6修复完成：改进变量类型推断，解决赋值/初始化类型不匹配

**修改文件**: [`src/backend/cgen/cgen.c`](src/backend/cgen/cgen.c)

#### 修改内容

1. **新增 [`is_pointer_like_type()`](src/backend/cgen/cgen.c:140) 辅助函数**
   - 统一检测指针类型（CN_TYPE_POINTER）和字符串类型（CN_TYPE_STRING，即char*）
   - 用于ADD/SUB指针算术类型推断

2. **ADD/SUB指针算术类型推断（行2603-2660）**
   - 移除`!new_type`限制：当dest.type已设为INT时，仍允许指针类型覆盖
   - 添加IMM_STR操作数视为char*类型的支持
   - 指针+整数=指针，整数+指针=指针，指针-整数=指针
   - 修复"char* + int 赋值给 long long int"的类型不匹配错误

3. **MOV指令类型传播增强（行2365-2460）**
   - 移除`!new_type`限制：允许指针/结构体类型覆盖INT/CHAR/BOOL等低精度类型
   - 添加IMM_STR→char*类型传播
   - 添加从ALLOCA映射表/函数参数/全局作用域获取符号类型的完整路径
   - 添加从reg_types获取寄存器类型的路径

4. **DEREF指令类型推断（行2662-2680）**
   - 解引用*ptr的结果类型是指针指向的类型
   - STRING类型(char*)解引用得到char

5. **STORE指令类型传播到ALLOCA变量增强（行2838-2910）**
   - 扩展源类型获取：支持REG、IMM_STR、src1.type三种来源
   - 允许POINTER类型覆盖STRING类型（char* → char**）
   - 允许POINTER类型覆盖STRUCT类型（struct X → struct X*）
   - 允许POINTER类型覆盖ENUM类型

6. **ALLOCA类型回写增强（行2985-3035）**
   - 允许POINTER类型覆盖STRING/STRUCT/ENUM类型的ALLOCA指令
   - 确保变量声明使用正确的类型字符串
