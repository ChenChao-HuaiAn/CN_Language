### 修复代码生成器结构体前向声明顺序问题

**时间**: 2026-04-06

#### 问题描述
在编译CN语言代码时，当结构体A通过指针引用结构体B，但B的定义在A之后时，生成的C代码会缺少结构体B的前向声明，导致C编译器报错 `incomplete type` 或 `unknown type name`。

#### 修复内容

**修改文件**: [`src/backend/cgen/cgen.c`](src/backend/cgen/cgen.c)

1. **添加前向声明** (第127-132行)
   - 添加 `is_type_already_generated` 和 `mark_type_as_generated` 函数的前向声明

2. **修改 `cn_cgen_struct_decl_with_prefix` 函数** (第1514-1576行)
   - 在生成结构体定义前，先检查所有字段类型
   - 对于指针类型字段，如果指向结构体，则输出前向声明
   - 使用 `is_type_already_generated` 避免重复声明

3. **修改 `cn_cgen_struct_type_definition_with_forward` 函数** (第1980-2036行)
   - 同样添加指针字段的前向声明处理逻辑

#### 测试验证

**测试用例**: [`tests/测试结构体前向声明.cn`](tests/测试结构体前向声明.cn)
```cn
结构体 A {
    B* 指针;
};

结构体 B {
    整数 值;
};
```

**生成的C代码**:
```c
// CN Language Global Struct Definitions
struct B;
struct A {
    struct B* 指针;
};

struct B {
    long long 值;
};
```

#### 验证结果
- ✅ CN源代码编译成功
- ✅ 生成的C代码包含正确的前向声明
- ✅ C代码编译成功（仅有未初始化变量警告，非错误）
