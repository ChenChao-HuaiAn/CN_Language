## 功能完善总结

### 修复IR生成器函数调用返回值处理问题

**问题描述**：函数调用返回值未正确赋值，导致生成的C代码中条件判断显示 `/* NONE */`

**根本原因**：在 `semantic_passes.c` 中，`infer_function_return_type` 函数在没有找到 return 语句时返回 void，然后这个 void 覆盖了函数声明中显式指定的返回类型。

**修复内容**：
1. **[`src/semantics/checker/semantic_passes.c`](src/semantics/checker/semantic_passes.c:357)** - 修改返回类型推断逻辑，只有当函数没有显式指定返回类型（或返回类型是 UNKNOWN）时，才使用推断的返回类型

**修复前代码**：
```c
// 推断函数返回类型：遍历函数体中的return语句
CnType *inferred_return_type = infer_function_return_type(fn_scope, fn->body, diagnostics);
if (inferred_return_type) {
    // 更新函数符号的返回类型
    CnSemSymbol *fn_sym = cn_sem_scope_lookup(global_scope, fn->name, fn->name_length);
    if (fn_sym && fn_sym->type && fn_sym->type->kind == CN_TYPE_FUNCTION) {
        fn_sym->type->as.function.return_type = inferred_return_type;
    }
}
```

**修复后代码**：
```c
// 推断函数返回类型：遍历函数体中的return语句
// 只有当函数没有显式指定返回类型，或返回类型是UNKNOWN时，才使用推断的返回类型
CnType *declared_return_type = fn->return_type;
if (!declared_return_type || declared_return_type->kind == CN_TYPE_UNKNOWN) {
    CnType *inferred_return_type = infer_function_return_type(fn_scope, fn->body, diagnostics);
    if (inferred_return_type) {
        // 更新函数符号的返回类型
        CnSemSymbol *fn_sym = cn_sem_scope_lookup(global_scope, fn->name, fn->name_length);
        if (fn_sym && fn_sym->type && fn_sym->type->kind == CN_TYPE_FUNCTION) {
            fn_sym->type->as.function.return_type = inferred_return_type;
        }
    }
}
```

**验证结果**：
- 修复前：`是同步点(r0, 0); if (/* NONE */) goto ...`
- 修复后：`r1 = 是同步点(r0, 0); if (r1) goto if_then_27; else goto if_merge_28;`

- [`src/semantics/resolution/scope_builder.c`](src/semantics/resolution/scope_builder.c) - 移除函数类型创建调试输出
- [`src/semantics/checker/semantic_passes.c`](src/semantics/checker/semantic_passes.c) - 移除标识符和函数调用调试输出
- [`src/ir/gen/irgen.c`](src/ir/gen/irgen.c) - 移除IR生成调试输出
- [`src/frontend/parser/parser.c`](src/frontend/parser/parser.c) - 移除解析器调试输出
