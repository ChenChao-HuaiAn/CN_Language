# AI 任务完成检查清单

> 本检查清单用于确保 AI 在调用 `attempt_completion` 之前完成所有必要步骤。
> AI 必须在每次任务完成前执行此检查清单验证。

---

## 🔍 任务完成前检查清单

在调用 `attempt_completion` 之前，AI **必须** 执行以下检查：

### 1. 功能完成检查

- [ ] 所有子任务已完成
- [ ] 代码已编写并通过编译
- [ ] 所有测试已通过（或已明确说明跳过原因）

### 2. 文档更新检查

- [ ] **TODO文件已更新**：相关TODO文件中的任务复选框已标记为 `[x]`
- [ ] **更新日志已更新**：`更新日志.md` 文件已更新本次修改内容
- [ ] **lessons.md已更新**：如有错误或经验教训，已记录到 `lessons.md`

### 3. Git 提交检查

- [ ] 提交信息使用中文编写
- [ ] 提交内容清晰说明了修改内容
- [ ] 已推送到 GitCode 和 GitHub 两个远程仓库

### 4. 代码质量检查

- [ ] 每个源文件不超过500行
- [ ] 每个函数不超过100行
- [ ] 代码添加了中文注释
- [ ] 遵循单一职责原则

---

## ⚠️ 强制执行规则

**如果存在未完成的检查项，AI 不得调用 `attempt_completion`，不得声称"任务完成"。**

### 检查流程

```
1. 完成代码实现
2. 运行测试验证
3. ✅ 更新TODO文件（标记所有完成的任务）
4. ✅ 更新更新日志.md
5. ✅ 记录错误到lessons.md（如有）
6. Git提交和推送
7. 再次确认所有检查项完成
8. 调用 attempt_completion
```

---

## 📝 常见遗漏项

| 遗漏项 | 后果 | 解决方案 |
|--------|------|----------|
| TODO文件未更新 | 任务状态不清晰 | 在attempt_completion前搜索 `- [ ]` 确认 |
| 更新日志未更新 | 无法追踪变更 | 提交前必须更新 |
| lessons.md未记录错误 | 重复犯错 | 出现错误立即记录 |
