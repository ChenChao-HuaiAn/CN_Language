# 模块加载器 (Module Loader)

## 概述

模块加载器负责解析和管理 CN_Language 的模块系统，包括模块导入、导出和可见性控制。模块系统允许代码组织和重用，同时提供命名空间隔离。

## 文件位置

[`src/frontend/module_loader/module_loader.c`](../../src/frontend/module_loader/module_loader.c:1)

## 模块系统概念

### 模块声明

```cn
// 模块文件必须以模块声明开始
模块 模块名;

// 示例
模块 数学模块;
模块 工具。字符串处理;
模块 图形。二维。点;
```

### 模块路径

模块使用点分路径表示层级关系：

```
模块路径：图形。二维。点
文件路径：图形/二维/点.cn
```

### 可见性

| 可见性 | 描述 | 关键字 |
|--------|------|--------|
| **私有** | 仅模块内可见 | `#私有` 或默认 |
| **公开** | 对所有模块可见 | `#公开` |
| **导出** | 显式导出符号 | `导出` |

## 核心数据结构

### CnModule 结构

```c
typedef struct CnModule {
    char *name;                  // 模块名称
    char *file_path;             // 文件路径
    CnAstProgram *program;       // AST 程序
    CnHashMap *exports;          // 导出符号表
    CnArray *imports;            // 导入列表
    CnModule **dependencies;     // 依赖模块
    size_t dependency_count;     // 依赖数量
    int is_loaded;               // 是否已加载
    int load_order;              // 加载顺序
} CnModule;
```

### CnModuleLoader 结构

```c
typedef struct CnModuleLoader {
    CnHashMap *loaded_modules;   // 已加载模块缓存
    CnArray *module_search_paths; // 模块搜索路径
    CnDiagnostics *diagnostics;  // 诊断信息
    CnAstProgram *current_program; // 当前程序
} CnModuleLoader;
```

## 导入语法

### 完整导入

```cn
// 导入整个模块
导入 数学模块;

// 使用完全限定名
整数 结果 = 数学模块。加法 (5, 3);
```

### 选择性导入

```cn
// 从模块导入特定符号
从 数学模块 导入 加法，减法，乘法;

// 直接使用符号
整数 和 = 加法 (10, 20);
```

### 别名导入

```cn
// 导入时重命名
从 数学模块 导入 乘法 作为 乘;
从 图形模块 导入 点 作为 图形点;

// 使用别名
整数 积 = 乘 (5, 6);
```

### 嵌套模块导入

```cn
// 导入嵌套模块
导入 图形。二维;
导入 图形。三维。向量;

// 选择性导入嵌套模块
从 图形。二维 导入 点，线，矩形;
```

## 导出语法

### 导出函数

```cn
模块 数学模块;

// 默认私有（不导出）
函数 内部辅助 () {
    // 只在模块内可见
}

// 显式导出
导出 函数 加法 (整数 a, 整数 b) {
    返回 a + b;
}

导出 函数 减法 (整数 a, 整数 b) {
    返回 a - b;
}
```

### 导出结构体

```cn
模块 数据结构;

// 导出结构体
导出 结构体 链表节点 {
    整数 数据;
    链表节点* 下一个;
}

// 导出结构体方法
导出 函数 创建节点 (整数 数据) {
    // ...
}
```

### 导出枚举

```cn
模块 常量;

// 导出枚举
导出 枚举 错误码 {
    成功，
    失败，
    无效参数，
    内存不足
}
```

### 批量导出

```cn
模块 工具模块;

#公开  // 设置默认可见性为公开

函数 工具函数 1() { }  // 自动导出
函数 工具函数 2() { }  // 自动导出

#私有  // 恢复默认为私有

函数 内部函数 () { }  // 不导出
```

## 模块解析流程

```
1. 解析模块声明
   ↓
2. 计算模块文件路径
   ↓
3. 检查模块缓存
   ↓
4. 读取并解析源文件
   ↓
5. 构建模块 AST
   ↓
6. 收集导出符号
   ↓
7. 处理依赖关系
   ↓
8. 缓存模块
```

## 循环依赖处理

```c
// 检测循环依赖
int detect_circular_dependency(CnModuleLoader *loader, CnModule *module)
{
    // 使用 DFS 检测循环
    CnHashSet *visited = create_hash_set();
    return has_cycle(loader, module, visited);
}

// 1. 报错并终止
```

## 错误处理

### 常见错误

| 错误 | 描述 | 示例 |
|------|------|------|
| 模块未找到 | 模块文件不存在 | `导入 不存在的模块;` |
| 循环依赖 | 模块 A 导入 B，B 导入 A | A.cn ↔ B.cn |
| 符号未导出 | 尝试导入未导出的符号 | `从 模块 导入 私有函数;` |
| 命名冲突 | 导入的符号名称冲突 | 从两个模块导入同名函数 |
| 模块加载失败 | 模块文件解析错误 | 语法错误 |

### 错误报告

```c
static void report_module_error(CnModuleLoader *loader, 
                                CnDiagCode code, 
                                const char *message,
                                const char *module_name)
{
    if (!loader || !loader->diagnostics) {
        return;
    }

    cn_support_diagnostics_report(loader->diagnostics,
                                  CN_DIAG_SEVERITY_ERROR,
                                  code,
                                  module_name,
                                  0, 0,
                                  message);
}
```

## 使用示例

### 模块定义

```cn
// 文件：数学/基础.cn
模块 数学。基础;

// 内部函数（私有）
函数 验证参数 (整数 a, 整数 b) {
    返回 a >= 0 && b >= 0;
}

// 导出函数（公开）
导出 函数 加法 (整数 a, 整数 b) {
    如果 (!验证参数 (a, b)) {
        返回 -1;  // 错误
    }
    返回 a + b;
}

导出 函数 减法 (整数 a, 整数 b) {
    返回 a - b;
}
```

### 模块使用

```cn
// 文件：主程序.cn
模块 主程序;

// 导入方式 1：完整导入
导入 数学。基础;

函数 使用数学 1() {
    整数 结果 = 数学。基础。加法 (5, 3);
}

// 导入方式 2：选择性导入
从 数学。基础 导入 加法，减法;

函数 使用数学 2() {
    整数 和 = 加法 (10, 20);
    整数 差 = 减法 (50, 30);
}

// 导入方式 3：别名导入
从 数学。基础 导入 加法 作为 加;

函数 使用数学 3() {
    整数 结果 = 加 (100, 200);
}
```

## 性能优化

1. **模块缓存**：已加载模块缓存在内存中，避免重复加载
2. **懒加载**：只在首次使用时加载模块
3. **并行加载**：无依赖关系的模块可以并行加载
4. **增量编译**：只重新编译变化的模块
