# 优化 Passes

## 概述

优化 Passes 是对 IR 进行变换以提高生成代码性能的过程。CN_Language 采用模块化 pass 设计，每个 pass 独立实现特定的优化策略。

## 文件位置

- [`src/ir/passes/constant_folding.c`](../../src/ir/passes/constant_folding.c:1) - 常量折叠
- [`src/ir/passes/dead_code_elimination.c`](../../src/ir/passes/dead_code_elimination.c:1) - 死代码消除

```c
typedef void (*CnIrPassFn)(CnIrModule *module);

// 注册 pass
void cn_ir_register_pass(CnIrPassFn pass);

void cn_ir_run_pass(CnIrPassFn pass, CnIrModule *module);
```

## 常量折叠 (Constant Folding)

### 概述

常量折叠在编译时计算常量表达式，减少运行时计算开销。

### 文件位置

[`src/ir/passes/constant_folding.c`](../../src/ir/passes/constant_folding.c:1)

### 实现原理

```c
// 检查操作数是否为常量
static bool is_constant(CnIrOperand op) {
    return op.kind == CN_IR_OP_IMM_INT;
}

// 折叠单条指令
static void fold_inst(CnIrInst *inst) {
    // 算术和比较运算
    if (inst->kind >= CN_IR_INST_ADD && inst->kind <= CN_IR_INST_GE) {
        if (is_constant(inst->src1) && is_constant(inst->src2)) {
            long long v1 = inst->src1.as.imm_int;
            long long v2 = inst->src2.as.imm_int;
            long long result = 0;
            bool foldable = true;

            switch (inst->kind) {
                case CN_IR_INST_ADD: result = v1 + v2; break;
                case CN_IR_INST_SUB: result = v1 - v2; break;
                case CN_IR_INST_MUL: result = v1 * v2; break;
                case CN_IR_INST_DIV: 
                    if (v2 != 0) result = v1 / v2; 
                    else foldable = false; 
                    break;
                case CN_IR_INST_MOD: 
                    if (v2 != 0) result = v1 % v2; 
                    else foldable = false; 
                    break;
                case CN_IR_INST_AND: result = v1 & v2; break;
                case CN_IR_INST_OR:  result = v1 | v2; break;
                case CN_IR_INST_XOR: result = v1 ^ v2; break;
                case CN_IR_INST_SHL: result = v1 << v2; break;
                case CN_IR_INST_SHR: result = v1 >> v2; break;
                case CN_IR_INST_EQ:  result = v1 == v2; break;
                case CN_IR_INST_NE:  result = v1 != v2; break;
                case CN_IR_INST_LT:  result = v1 < v2; break;
                case CN_IR_INST_LE:  result = v1 <= v2; break;
                case CN_IR_INST_GT:  result = v1 > v2; break;
                case CN_IR_INST_GE:  result = v1 >= v2; break;
                default: foldable = false; break;
            }

            if (foldable) {
                // 将指令替换为 MOV 立即数
                inst->kind = CN_IR_INST_MOV;
                inst->src1.kind = CN_IR_OP_IMM_INT;
                inst->src1.as.imm_int = result;
                inst->src2.kind = CN_IR_OP_NONE;
            }
        }
    } 
    // 一元运算
    else if (inst->kind == CN_IR_INST_NEG || inst->kind == CN_IR_INST_NOT) {
        if (is_constant(inst->src1)) {
            long long v1 = inst->src1.as.imm_int;
            long long result = (inst->kind == CN_IR_INST_NEG) ? -v1 : !v1;
            inst->kind = CN_IR_INST_MOV;
            inst->src1.as.imm_int = result;
        }
    }
}

// Pass 入口函数
void cn_ir_pass_constant_folding(CnIrModule *module) {
    if (!module) return;
    
    for (CnIrFunction *func = module->first_func; func; func = func->next) {
        for (CnIrBasicBlock *block = func->first_block; block; block = block->next) {
            for (CnIrInst *inst = block->first_inst; inst; inst = inst->next) {
                fold_inst(inst);
            }
        }
    }
}
```

### 优化示例

**优化前：**
```ir
%1 = MOV 10
%2 = MOV 20
%3 = ADD %1, %2
%4 = MUL %3, 2
RET %4
```

**优化后：**
```ir
%1 = MOV 60
RET %1
```

### 边界情况处理

1. **除零检查**：除法运算时检查除数是否为零
2. **溢出检查**：大数运算可能溢出（当前实现未处理）
3. **浮点常量**：当前仅支持整数常量折叠

## 死代码消除 (Dead Code Elimination)

### 概述

死代码消除移除永远不会执行的代码，减小代码体积并提高性能。

### 文件位置

[`src/ir/passes/dead_code_elimination.c`](../../src/ir/passes/dead_code_elimination.c:1)

### 实现原理

```c
// 标记可达块（DFS）
static void mark_reachable(CnIrBasicBlock *block, 
                           bool *reachable, 
                           CnIrBasicBlock **blocks, 
                           int count) {
    // 查找块的索引
    int idx = -1;
    for (int i = 0; i < count; i++) {
        if (blocks[i] == block) {
            idx = i;
            break;
        }
    }
    
    if (idx == -1 || reachable[idx]) return;
    
    // 标记为可达
    reachable[idx] = true;
    
    // 递归标记后继块
    for (CnIrBasicBlockList *succ = block->succs; succ; succ = succ->next) {
        mark_reachable(succ->block, reachable, blocks, count);
    }
}

// Pass 入口函数
void cn_ir_pass_dead_code_elimination(CnIrModule *module) {
    if (!module) return;

    for (CnIrFunction *func = module->first_func; func; func = func->next) {
        if (!func->first_block) continue;

        // 1. 收集所有基本块
        int block_count = 0;
        for (CnIrBasicBlock *b = func->first_block; b; b = b->next) {
            block_count++;
        }

        CnIrBasicBlock **blocks = malloc(sizeof(CnIrBasicBlock *) * block_count);
        bool *reachable = calloc(block_count, sizeof(bool));
        
        int i = 0;
        for (CnIrBasicBlock *b = func->first_block; b; b = b->next) {
            blocks[i++] = b;
        }

        // 2. 从入口块开始标记可达性
        mark_reachable(func->first_block, reachable, blocks, block_count);

        // 3. 移除不可达块
        for (int j = 0; j < block_count; j++) {
            if (!reachable[j]) {
                CnIrBasicBlock *b = blocks[j];
                
                // 从函数链表中移除
                if (b->prev) {
                    b->prev->next = b->next;
                } else {
                    func->first_block = b->next;
                }

                if (b->next) {
                    b->next->prev = b->prev;
                } else {
                    func->last_block = b->prev;
                }
            }
        }

        free(blocks);
        free(reachable);
    }
}
```

### 优化示例

**优化前：**
```ir
define 整数 @测试 () {
entry:
    JMP exit
    
unreachable:
    %1 = ADD 1, 2
    %2 = MUL %1, 3
    RET %2
    
exit:
    RET 0
}
```

**优化后：**
```ir
define 整数 @测试 () {
entry:
    JMP exit
    
exit:
    RET 0
}
```

### 边界情况处理

1. **循环结构**：正确处理循环的可达性
2. **异常处理**：考虑异常路径的可达性
3. **多重出口**：处理多个返回点的情况

## Pass 管理器

### 默认 Pass 序列

```c
void cn_ir_run_default_passes(CnIrModule *module) {
    // 1. 常量折叠
    cn_ir_pass_constant_folding(module);
    
    // 2. 死代码消除
    cn_ir_pass_dead_code_elimination(module);
    
    // 可以添加更多 passes...
}
```

### Pass 执行顺序

```
IR 生成 → [常量折叠] → [死代码消除] → [其他 passes] → 代码生成
```

### Pass 依赖

某些 pass 可能有依赖关系：

```c
// 示例：常量传播依赖常量折叠
void cn_ir_run_optimization_pipeline(CnIrModule *module) {
    cn_ir_pass_constant_folding(module);
    cn_ir_pass_constant_propagation(module);  // 依赖常量折叠
    cn_ir_pass_dead_code_elimination(module); // 在常量传播后消除死代码
}
```

## 性能考虑

1. **Pass 复杂度**：每个 pass 应该是 O(n) 或 O(n log n)
2. **迭代优化**：某些优化需要多次迭代才能达到不动点
3. **内存使用**：避免在 pass 中创建过多临时对象
