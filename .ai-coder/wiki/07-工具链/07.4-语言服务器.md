# 语言服务器 (cnlsp)

## 概述

`cnlsp` 是 CN_Language 的语言服务器协议 (LSP) 实现，为编辑器提供智能代码补全、跳转定义、查找引用等功能。

## 使用方法

### 启动语言服务器

```bash
# 标准输入/输出模式
cnlsp --stdio

# TCP 模式
cnlsp --tcp --port 20837
```

### VSCode 配置

```json
// .vscode/settings.json
{
    "cnlang.lsp.command": "cnlsp",
    "cnlang.lsp.args": ["--stdio"]
}
```

## 支持的功能

### 代码补全

```cn
// 输入 "打印" 后自动补全
打印 (|)  // 光标位置显示参数提示

// 结构体成员补全
点 p;
p.|  // 显示 x, y 等成员
```

### 跳转定义

```cn
函数 加法 (整数 a, 整数 b) {  // 按 F12 跳转到这里
    返回 a + b;
}

函数 主程序 () {
    整数 结果 = 加法 (5, 3);  // 按 F12 跳转到定义
}
```

### 查找引用

```cn
// 在函数名上按 Shift+F12 查找所有引用
函数 目标函数 () { }  // 引用位置会高亮显示
```

### 悬停提示

```cn
// 鼠标悬停显示类型信息
函数 主程序 () {
    整数 x = 10;  // 悬停显示：整数
}
```

### 符号搜索

```cn
// Ctrl+T 搜索工作区符号
// 输入函数名、变量名等快速定位
```

### 重命名

```cn
// F2 重命名符号（所有引用同步更新）
整数 旧名称 = 10;  // 重命名为 新名称
```

### 诊断信息

```cn
// 实时显示错误和警告
整数 x = "文本";  // 显示类型错误
```

## LSP 协议

### 初始化

```json
// 客户端 -> 服务器
{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {
        "processId": 12345,
        "rootUri": "file:///path/to/project",
        "capabilities": {
            "textDocument": {
                "completion": {},
                "hover": {},
                "definition": {}
            }
        }
    }
}
```

### 代码补全

```json
// 客户端 -> 服务器
{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "textDocument/completion",
    "params": {
        "textDocument": {"uri": "file:///path/to/file.cn"},
        "position": {"line": 5, "character": 10}
    }
}

// 服务器 -> 客户端
{
    "jsonrpc": "2.0",
    "id": 2,
    "result": [
        {
            "label": "打印",
            "kind": 3,
            "detail": "void 打印 (字符串)"
        }
    ]
}
```

### 跳转定义

```json
// 客户端 -> 服务器
{
    "jsonrpc": "2.0",
    "id": 3,
    "method": "textDocument/definition",
    "params": {
        "textDocument": {"uri": "file:///path/to/file.cn"},
        "position": {"line": 5, "character": 10}
    }
}

// 服务器 -> 客户端
{
    "jsonrpc": "2.0",
    "id": 3,
    "result": {
        "uri": "file:///path/to/file.cn",
        "range": {
            "start": {"line": 0, "character": 0},
            "end": {"line": 0, "character": 20}
        }
    }
}
```

## 文档管理

### 打开文档

```c
void lsp_handle_did_open(CnLspServer *server, CnLspMessage *msg) {
    CnDocument *doc = document_manager_open(server->doc_manager,
                                             msg->params.uri,
                                             msg->params.text);
    parse_document(doc);
}
```

### 修改文档

```c
void lsp_handle_did_change(CnLspServer *server, CnLspMessage *msg) {
    CnDocument *doc = document_manager_get(server->doc_manager, msg->params.uri);
    document_manager_apply_changes(doc, msg->params.contentChanges);
    reparse_document(doc);
    publish_diagnostics(doc);
}
```

### 关闭文档

```c
void lsp_handle_did_close(CnLspServer *server, CnLspMessage *msg) {
    document_manager_close(server->doc_manager, msg->params.uri);
}
```

## 诊断发布

```c
void publish_diagnostics(CnDocument *doc) {
    CnDiagnostics *diagnostics = analyze_document(doc);
    
    CnLspMessage msg;
    msg.method = "textDocument/publishDiagnostics";
    msg.params.uri = doc->uri;
    msg.params.diagnostics = convert_to_lsp_diagnostics(diagnostics);
    
    lsp_send_notification(msg);
}
```

## 性能优化

### 增量解析

```c
// 只重新解析修改的部分
void incremental_parse(CnDocument *doc, CnTextRange change) {
    CnAstNode *affected = find_affected_nodes(doc->ast, change);
    reparse_nodes(affected);
    update_ast(doc->ast, affected);
}
```

### 缓存

```c
// 缓存符号表和类型信息
CnSymbolTable *get_cached_symbols(CnDocument *doc) {
    if (doc->symbols_dirty) {
        doc->symbols = build_symbols(doc);
        doc->symbols_dirty = false;
    }
    return doc->symbols;
}
```
