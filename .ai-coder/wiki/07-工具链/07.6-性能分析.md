# 性能分析 (cnperf)

## 概述

`cnperf` 是 CN_Language 的性能分析工具，用于测量和分析程序的执行性能。

## 文件位置

[`src/cli/cnperf/main.c`](../../src/cli/cnperf/main.c:1)
[`src/support/perf/perf.c`](../../src/support/perf/perf.c:1)

## 使用方法

### 基本用法

```bash
# 分析程序执行
cnperf 程序.cn

# 分析已编译的程序
cnperf ./程序

# 输出详细报告
cnperf --verbose 程序.cn

# 输出 JSON 格式
cnperf --json 程序.cn
```

### 命令行选项

| 选项 | 描述 |
|------|------|
| `--verbose` | 详细输出 |
| `--json` | JSON 格式输出 |
| `--output <文件>` | 输出到文件 |
| `--hotspot <n>` | 显示前 n 个热点 |
| `--call-graph` | 生成调用图 |
| `-h`, `--help` | 显示帮助 |

## 调用图

### 生成调用图

```bash
cnperf --call-graph 程序.cn --output callgraph.dot
```

### 可视化调用图

```bash
# 转换为 PNG
dot -Tpng callgraph.dot -o callgraph.png

# 转换为 SVG
dot -Tsvg callgraph.dot -o callgraph.svg
```

### 调用图示例


## 性能优化建议

### 自动建议

```bash
cnperf --suggest 程序.cn
```

```
性能优化建议:

1. 函数 '密集计算' 占用了 45% 的执行时间
   建议：考虑使用内联或算法优化

2. 在循环中有频繁的内存分配 (1000 次)
   建议：使用内存池或预分配

3. 字符串连接操作频繁
   建议：使用字符串构建器或预分配缓冲区

4. 函数 '辅助函数' 被调用 10000 次但只执行简单操作
   建议：考虑内联该函数
```
