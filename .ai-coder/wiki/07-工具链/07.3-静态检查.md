# 静态检查 (cncheck)

## 概述

`cncheck` 是 CN_Language 的静态代码分析工具，在不运行代码的情况下发现潜在问题。

## 文件位置

[`src/cli/cncheck/main.c`](../../src/cli/cncheck/main.c:1)
[`src/analysis/checker/static_check.c`](../../src/analysis/checker/static_check.c:1)

## 使用方法

### 基本用法

```bash
# 检查单个文件
cncheck 文件.cn

# 检查多个文件
cncheck 文件 1.cn 文件 2.cn

# 递归检查目录
cncheck -r 目录/

# 输出 JSON 格式
cncheck --json 文件.cn

# 忽略特定警告
cncheck --ignore=unused-variable 文件.cn
```

### 命令行选项

| 选项 | 描述 |
|------|------|
| `-r`, `--recursive` | 递归检查目录 |
| `--json` | 输出 JSON 格式 |
| `--ignore=<规则>` | 忽略特定规则 |
| `--only=<规则>` | 只检查特定规则 |
| `--fail-on=<级别>` | 在指定级别失败 |
| `-h`, `--help` | 显示帮助 |

## 检查规则

### 未使用变量

```cn
// 警告：变量 'x' 未使用
函数 测试 () {
    整数 x = 10;
    打印 ("hello");
}
```

### 未初始化变量

```cn
// 警告：变量 'x' 可能未初始化就被使用
函数 测试 () {
    整数 x;
    如果 (假) {
        x = 10;
    }
    打印 (x);  // 可能使用未初始化的值
}
```

### 类型转换警告

```cn
// 警告：隐式类型转换可能丢失精度
函数 测试 () {
    小数 pi = 3.14;
    整数 x = pi;  // 丢失小数部分
}
```

### 空指针检查

```cn
// 警告：可能的空指针解引用
函数 测试 (整数* 指针) {
    打印 (*指针);  // 指针可能为空
}
```

### 死代码检测

```cn
// 警告：不可达代码
函数 测试 () {
    返回;
    打印 ("这行代码永远不会执行");  // 死代码
}
```

### 循环边界检查

```cn
// 警告：可能的数组越界
函数 测试 () {
    整数 数组 [10];
    循环 (整数 i = 0; i <= 10; i = i + 1) {  // 应该是 < 10
        数组 [i] = i;
    }
}
```

### 资源泄漏

```cn
// 警告：可能的资源泄漏
函数 测试 () {
    文件* f = 文件。打开 ("test.txt", "r");
    // 忘记关闭文件
}
```

## 检查级别

### 错误 (Error)

必须修复的问题：
- 语法错误
- 类型错误
- 未定义符号

### 警告 (Warning)

建议修复的问题：
- 未使用变量
- 隐式类型转换
- 可能的空指针

### 提示 (Info)

代码风格建议：
- 命名约定
- 代码格式
- 注释缺失

## 配置

### 配置文件

```json
// .cncheck.json
{
    "rules": {
        "unused-variable": "warning",
        "uninitialized-variable": "error",
        "implicit-cast": "warning",
        "null-dereference": "error"
    },
    "ignores": [
        "**/generated/*.cn",
        "**/third_party/*.cn"
    ],
    "maxLineLength": 100,
    "checkStyle": true
}
```

### 行内抑制

```cn
// 抑制单行警告
整数 x = 未使用 ();  // cncheck-ignore: unused-variable

// 抑制代码块警告
#cncheck-disable unused-variable
函数 测试 () {
    整数 x = 10;
}
#cncheck-enable unused-variable
```

## 输出格式

### 默认格式

```
文件.cn:10:5: 警告：变量 'x' 未使用 [-Wunused-variable]
    整数 x = 10;
        ^
```

### JSON 格式

```json
{
    "file": "文件.cn",
    "line": 10,
    "column": 5,
    "severity": "warning",
    "message": "变量 'x' 未使用",
    "rule": "unused-variable"
}
```

## 集成

### VSCode 集成

```json
// .vscode/settings.json
{
    "cncheck.enable": true,
    "cncheck.rules": {
        "unused-variable": "warning"
    }
}
```

### CI/CD 集成

```yaml
# .github/workflows/check.yml
name: Static Check
on: [push, pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Install cncheck
        run: cmake --build build --target cncheck
      - name: Run check
        run: cncheck src/**/*.cn
```
