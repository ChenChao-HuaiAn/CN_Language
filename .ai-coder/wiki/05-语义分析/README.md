# 语义分析文档

## 概述

语义分析是编译器前端的最后一个阶段，负责验证程序的含义是否正确。语义分析包括名称解析、类型检查和语义验证。

## 模块结构

```
src/semantics/
├── symbols/              # 符号表和类型系统
│   ├── symbol_table.c   # 符号表实现
│   └── type_system.c    # 类型系统实现
├── resolution/           # 名称解析
│   ├── scope_builder.c  # 作用域构建
│   └── module_semantics.c # 模块语义
└── checker/              # 语义检查
    ├── semantic_passes.c # 语义检查 passes
    └── freestanding_check.c # Freestanding 检查
```

## 语义分析流程

```
AST → [作用域构建] → [名称解析] → [类型检查] → [语义验证] → IR 生成
```
