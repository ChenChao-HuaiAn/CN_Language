# 调试指南

## 概述

本文档介绍 CN_Language 项目的调试方法和技巧。

## 编译调试版本

### 基本调试配置

```bash
# 配置调试版本
cmake -B build -DCMAKE_BUILD_TYPE=Debug

# 编译
cmake --build build
```

### 带消毒器的调试版本

```bash
# AddressSanitizer (内存错误)
cmake -B build -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_C_FLAGS="-fsanitize=address -fno-omit-frame-pointer"

# UndefinedBehaviorSanitizer (未定义行为)
cmake -B build -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_C_FLAGS="-fsanitize=undefined"

# MemorySanitizer (内存泄漏)
cmake -B build -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_C_FLAGS="-fsanitize=memory"

# ThreadSanitizer (数据竞争)
cmake -B build -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_C_FLAGS="-fsanitize=thread"
```

## GDB 调试

### 启动 GDB

```bash
# 启动 GDB
gdb ./build/src/cnc

# 带参数启动
gdb --args ./build/src/cnc 程序.cn
```

### 基本 GDB 命令

```gdb
# 设置断点
break main
break cn_frontend_parse_program
break file.c:100

# 运行程序
run
run 程序.cn

# 单步执行
next
step

# 继续执行
continue

# 打印变量
print variable
print/x variable  # 十六进制

# 查看栈帧
backtrace
frame 1

# 查看内存
x/10x address
```

### GDB Python 脚本

```python
# .gdbinit
set print pretty on
set pagination off

# 自定义 CN_Language 命令
define print-ast
    call cn_frontend_ast_print($arg0)
end

define print-ir
    call cn_ir_module_print($arg0)
end
```

## LLDB 调试

### 启动 LLDB

```bash
# 启动 LLDB
lldb ./build/src/cnc

# 带参数启动
lldb -- ./build/src/cnc 程序.cn
```

### 基本 LLDB 命令

```lldb
# 设置断点
breakpoint set --name main
breakpoint set --file parser.c --line 100

# 运行程序
run
run 程序.cn

# 单步执行
next
step

# 继续执行
continue

# 打印变量
print variable
frame variable

# 查看栈帧
thread backtrace
```

## Valgrind 调试

### 内存泄漏检测

```bash
# 运行 Valgrind
valgrind --leak-check=full ./build/src/cnc 程序.cn

# 详细输出
valgrind --leak-check=full --show-leak-kinds=all ./build/src/cnc 程序.cn

# 生成日志
valgrind --log-file=valgrind.log ./build/src/cnc 程序.cn
```

### 输出示例

```
==12345== 100 bytes in 1 blocks are definitely lost
==12345==    at 0x4C2FB55: malloc (vg_replace_malloc.c:299)
==12345==    by 0x401234: cn_frontend_parser_new (parser.c:105)
==12345==    by 0x401000: main (main.c:50)
```

## 日志调试

### 启用调试日志

```c
// 在代码中添加调试日志
#define DEBUG_PRINT(fmt, ...) \
    fprintf(stderr, "DEBUG: %s:%d:%s(): " fmt "\n", \
            __FILE__, __LINE__, __func__, ##__VA_ARGS__)

// 使用
DEBUG_PRINT("变量值：%d", value);
```

### 环境变量控制

```bash
# 设置调试级别
export CN_DEBUG_LEVEL=3

# 运行程序
./build/src/cnc 程序.cn
```

## 核心转储调试

### 启用核心转储

```bash
# 允许核心转储
ulimit -c unlimited

# 设置核心转储路径
echo "/tmp/core.%e.%p" > /proc/sys/kernel/core_pattern
```

### 分析核心转储

```bash
# 使用 GDB 分析
gdb ./build/src/cnc /tmp/core.cnc.12345

# 查看崩溃原因
(gdb) backtrace
(gdb) info registers
```

## 性能调试

### 使用 perf

```bash
# 性能分析
perf record ./build/src/cnc 程序.cn

# 查看报告
perf report

# 生成火焰图
perf script | stackcollapse-perf.pl | flamegraph.pl > flame.svg
```

### 使用 gprof

```bash
# 编译带 profiling
cmake -B build -DCMAKE_C_FLAGS="-pg"
cmake --build build

# 运行程序
./build/src/cnc 程序.cn

# 查看报告
gprof ./build/src/cnc gmon.out
```

## 常见问题调试

### 段错误

```bash
# 使用 GDB 定位
gdb --args ./build/src/cnc 程序.cn
(gdb) run
# 程序崩溃后
(gdb) backtrace
(gdb) frame 5
(gdb) print variable
```

### 内存泄漏

```bash
# 使用 AddressSanitizer
export ASAN_OPTIONS=detect_leaks=1
./build/src/cnc 程序.cn

# 或使用 Valgrind
valgrind --leak-check=full ./build/src/cnc 程序.cn
```

### 无限循环

```bash
# 附加到运行中的进程
gdb -p $(pgrep cnc)

# 查看当前执行位置
(gdb) backtrace

# 设置条件断点
(gdb) break parser.c:100 if i > 1000
```

### 类型错误

```bash
# 启用类型检查日志
export CN_DEBUG_TYPES=1
./build/src/cnc 程序.cn

# 或使用调试版本
cmake -B build -DCMAKE_BUILD_TYPE=Debug
```
