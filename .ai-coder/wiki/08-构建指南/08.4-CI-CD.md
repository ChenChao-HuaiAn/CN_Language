# CI/CD

## 概述

本文档介绍 CN_Language 项目的持续集成和持续部署配置。

## GitHub Actions

### 工作流配置

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        build_type: [Debug, Release]
    
    steps:
    - uses: actions/checkout@v3
    
    - name: 安装依赖
      run: |
        sudo apt update
        sudo apt install -y cmake gcc g++ ninja-build
    
    - name: 配置 CMake
      run: cmake -B build -DCMAKE_BUILD_TYPE=${{ matrix.build_type }}
    
    - name: 编译
      run: cmake --build build -j$(nproc)
    
    - name: 运行测试
      run: |
        cd build
        ctest --output-on-failure
```

### 代码覆盖率

```yaml
# .github/workflows/coverage.yml
name: Coverage

on:
  push:
    branches: [ main ]

jobs:
  coverage:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: 安装依赖
      run: |
        sudo apt update
        sudo apt install -y cmake gcc g++ gcovr
    
    - name: 配置 CMake
      run: |
        cmake -B build \
          -DCMAKE_BUILD_TYPE=Debug \
          -DCMAKE_C_FLAGS="--coverage"
    
    - name: 编译
      run: cmake --build build
    
    - name: 运行测试
      run: |
        cd build
        ctest --output-on-failure
    
    - name: 生成覆盖率报告
      run: |
        cd build
        gcovr -r .. --html --html-details -o coverage.html
    
    - name: 上传覆盖率报告
      uses: actions/upload-artifact@v3
      with:
        name: coverage-report
        path: build/coverage.html
```

### 发布流程

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
    
    steps:
    - uses: actions/checkout@v3
    
    - name: 配置 CMake
      run: cmake -B build -DCMAKE_BUILD_TYPE=Release
    
    - name: 编译
      run: cmake --build build
    
    - name: 打包
      run: |
        cd build
        cpack -G ZIP
    
    - name: 创建 Release
      uses: softprops/action-gh-release@v1
      with:
        files: build/*.zip
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## 本地 CI 脚本

### PowerShell 测试脚本

```powershell
# tests/run_all_tests.ps1

$ErrorActionPreference = "Stop"

Write-Host "=== CN_Language 测试套件 ===" -ForegroundColor Cyan

# 构建
Write-Host "`n[1/3] 构建项目..." -ForegroundColor Yellow
cmake --build build --config Release

# 运行单元测试
Write-Host "`n[2/3] 运行单元测试..." -ForegroundColor Yellow
Set-Location build
$testResult = ctest -C Release --output-on-failure
Set-Location ..

# 运行集成测试
Write-Host "`n[3/3] 运行集成测试..." -ForegroundColor Yellow
./tests/run_integration_tests.ps1

# 汇总结果
if ($LASTEXITCODE -eq 0) {
    Write-Host "`n所有测试通过！" -ForegroundColor Green
} else {
    Write-Host "`n测试失败！" -ForegroundColor Red
    exit 1
}
```

### Bash 测试脚本

## 预提交钩子

### Git 钩子

```bash
#!/bin/bash
# .git/hooks/pre-commit

set -e

echo "运行代码格式化检查..."
cnfmt --check src/**/*.cn

echo "运行静态检查..."
cncheck src/**/*.cn

echo "运行单元测试..."
cd build
ctest -R UnitTest

echo "预提交检查通过！"
```

### pre-commit 配置

```yaml
# .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      - id: cnfmt
        name: CN Format Check
        entry: cnfmt --check
        language: system
        files: \.cn$
      
      - id: cncheck
        name: CN Static Check
        entry: cncheck
        language: system
        files: \.cn$
      
      - id: cmake-build
        name: CMake Build
        entry: cmake --build build
        language: system
        pass_filenames: false
```

## Docker 集成

### Dockerfile

```dockerfile
# Dockerfile
FROM ubuntu:22.04

RUN apt update && apt install -y \
    cmake \
    gcc \
    g++ \
    ninja-build \
    git \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /workspace

COPY . .

RUN cmake -B build -DCMAKE_BUILD_TYPE=Release
RUN cmake --build build

CMD ["ctest", "--test-dir", "build", "--output-on-failure"]
```

### Docker Compose

```yaml
# docker-compose.yml
version: '3'

services:
  build:
    build: .
    volumes:
      - .:/workspace
  
  test:
    build: .
    command: ctest --test-dir build --output-on-failure
    volumes:
      - .:/workspace
```

## 质量门禁

### 覆盖率要求

```yaml
# .github/workflows/quality.yml
jobs:
  quality-gates:
    runs-on: ubuntu-latest
    
    steps:
    - name: 检查覆盖率
      run: |
        coverage=$(grep -oP '"line_coverage":\s*\K\d+' coverage.json)
        if [ "$coverage" -lt 80 ]; then
          echo "覆盖率不足：$coverage% < 80%"
          exit 1
        fi
```
