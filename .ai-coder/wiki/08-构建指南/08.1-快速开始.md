# 快速开始

## 概述

本文档指导您快速完成 CN_Language 项目的配置、构建和测试。

## 1. 获取源码

### 克隆仓库

```bash
git clone https://github.com/yourusername/CN_Language.git
cd CN_Language
```

### 更新子模块

```bash
git submodule update --init --recursive
```

## 2. 配置构建环境

### Windows (MSYS2)

```bash
# 安装 MSYS2
# 从 https://www.msys2.org/ 下载安装

# 在 MSYS2 UCRT64 终端中安装依赖
pacman -S mingw-w64-ucrt-x86_64-cmake
pacman -S mingw-w64-ucrt-x86_64-gcc
pacman -S mingw-w64-ucrt-x86_64-ninja

# 配置环境变量
export PATH=/ucrt64/bin:$PATH
```

### Windows (Visual Studio)

```bash
# 安装 Visual Studio 2019 或更高版本
# 安装 "使用 C++ 的桌面开发" 工作负载

# 在开发者命令提示符中配置
```

### Linux (Ubuntu/Debian)

```bash
# 安装依赖
sudo apt update
sudo apt install -y cmake gcc g++ ninja-build git

# 可选：安装调试工具
sudo apt install -y gdb valgrind
```

### Linux (Fedora/RHEL)

```bash
# 安装依赖
sudo dnf install -y cmake gcc gcc-c++ ninja-build git
```

### macOS

```bash
# 安装 Xcode 命令行工具
xcode-select --install

# 使用 Homebrew 安装依赖
brew install cmake ninja
```

## 3. 配置 CMake

### 基本配置

```bash
# 创建构建目录
cmake -B build

# 或使用 Ninja 生成器
cmake -B build -G Ninja
```

### 发布版本配置

```bash
# 发布版本（优化）
cmake -B build -DCMAKE_BUILD_TYPE=Release

# 带调试信息的发布版本
cmake -B build -DCMAKE_BUILD_TYPE=RelWithDebInfo
```

### 调试版本配置

```bash
# 调试版本
cmake -B build -DCMAKE_BUILD_TYPE=Debug

# 带地址消毒器的调试版本
cmake -B build -DCMAKE_BUILD_TYPE=Debug \
    -DCMAKE_C_FLAGS="-fsanitize=address"
```

### 完整配置选项

```bash
cmake -B build \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX=/usr/local \
    -DCN_BUILD_TESTS=ON \
    -DCN_BUILD_EXAMPLES=ON \
    -DCN_ENABLE_LTO=OFF
```

## 4. 编译

### 使用 Make

```bash
# 编译所有目标
cmake --build build

# 指定并行编译
cmake --build build -j$(nproc)

# 编译特定目标
cmake --build build --target cnc
cmake --build build --target cnfmt
```

### 使用 Ninja

```bash
# 编译所有目标
cmake --build build

# 指定并行编译
cmake --build build -j8
```

### Windows (Visual Studio)

```bash
# 使用 MSBuild
cmake --build build --config Release
```

## 5. 安装

### 系统范围安装

```bash
# Linux/macOS
sudo cmake --install build

# Windows (管理员权限)
cmake --install build
```

### 用户范围安装

```bash
# 安装到用户目录
cmake --install build --prefix ~/.local
```

### 打包

```bash
# 创建 CPack 包
cd build
cpack -G ZIP
```

## 6. 运行测试

### 运行所有测试

```bash
cd build
ctest --output-on-failure
```

### 运行特定测试

```bash
# 运行词法分析器测试
ctest -R Lexer --output-on-failure

# 运行前端测试
ctest -R "Frontend.*" --output-on-failure
```

### 并行测试

```bash
ctest -j4 --output-on-failure
```

### 详细输出

```bash
ctest -V
```

## 7. 验证安装

### 检查版本

```bash
cnc --version
cnfmt --version
cncheck --version
```

### 编译 Hello World

```cn
// hello.cn
函数 主程序 () {
    打印 ("你好，世界！\n");
    返回 0;
}
```

```bash
cnc hello.cn -o hello
./hello
```

## 8. 常见问题

### 问题：CMake 找不到编译器

**解决：**
```bash
# 确保编译器在 PATH 中
which gcc
which g++

# 或显式指定
cmake -B build -DCMAKE_C_COMPILER=gcc
```

### 问题：链接错误

**解决：**
```bash
# 清理并重新构建
rm -rf build
cmake -B build
cmake --build build
```

### 问题：测试失败

**解决：**
```bash
# 查看详细错误
ctest -V

# 运行单个测试文件
./build/tests/unit/lexer_test
```

### 问题：内存泄漏

**解决：**
```bash
# 使用 AddressSanitizer 构建
cmake -B build -DCMAKE_C_FLAGS="-fsanitize=address"
cmake --build build

# 运行程序
./build/src/cnc 程序.cn
```

## 9. 清理

### 清理构建产物

```bash
# 清理构建目录
rm -rf build

# 或只清理特定目标
cmake --build build --target clean
```

### 完全重置

```bash
# 删除所有生成的文件
git clean -fdx
git checkout .
```
