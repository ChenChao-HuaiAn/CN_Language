# 任务预检提示 (Pre-Check Prompt)

## 使用时机

在开始任何代码编写任务之前，执行此预检流程。

## 预检指令

```
你即将开始一个代码编写任务。在开始编码之前，请完成以下预检：

## Step 1: 任务规模评估

请预估以下指标：
- 代码量预估：_____ 行（限制：≤500行）
- 文件数预估：_____ 个（限制：≤10个文件）
- 预估执行时间：_____ 分钟（限制：≤30分钟）

## Step 2: 复杂度评估

请明确以下内容：
- 输入定义：[明确输入数据/参数]
- 输出定义：[明确输出结果/返回值]
- 边界条件：[列出边界场景]
- 异常场景：[列出可能的异常]

## Step 3: 预检决策

| 条件 | 处理方式 |
|------|----------|
| 代码量 > 500行 | 建议拆分为多个子任务 |
| 文件数 > 10个 | 建议拆分为多个子任务 |
| 复杂度不明确 | 先澄清需求再开始 |

## Step 4: 输出预检结果

```markdown
## 任务预检结果

### 规模评估
- 代码量：[X] 行 → ✅ 通过 / ❌ 需拆分
- 文件数：[X] 个 → ✅ 通过 / ❌ 需拆分
- 执行时间：[X] 分钟 → ✅ 通过

### 复杂度评估
- 输入定义：[已明确 / 需澄清]
- 输出定义：[已明确 / 需澄清]
- 边界条件：[已明确 / 需澄清]
- 异常场景：[已明确 / 需澄清]

### 预检结论
[ ] 通过 - 可以开始编码
[ ] 需拆分 - 建议拆分为以下子任务：
    - 子任务1: [描述]
    - 子任务2: [描述]
[ ] 需澄清 - 以下问题需要明确：
    - 问题1: [描述]
    - 问题2: [描述]
```
```

## 预检示例

### 示例 1：预检通过

```markdown
## 任务预检结果

### 规模评估
- 代码量：150 行 → ✅ 通过
- 文件数：3 个 → ✅ 通过
- 执行时间：15 分钟 → ✅ 通过

### 复杂度评估
- 输入定义：已明确 - 用户ID和查询参数
- 输出定义：已明确 - 用户信息DTO
- 边界条件：已明确 - 用户不存在、参数为空
- 异常场景：已明确 - 数据库连接失败

### 预检结论
[x] 通过 - 可以开始编码
```

### 示例 2：预检需拆分

```markdown
## 任务预检结果

### 规模评估
- 代码量：800 行 → ❌ 需拆分（超出500行限制）
- 文件数：15 个 → ❌ 需拆分（超出10个限制）
- 执行时间：60 分钟 → ❌ 需拆分（超出30分钟限制）

### 复杂度评估
- 输入定义：已明确
- 输出定义：已明确
- 边界条件：已明确
- 异常场景：已明确

### 预检结论
[ ] 需拆分 - 建议拆分为以下子任务：
    - 子任务1: 创建数据模型和接口定义（~100行，2个文件）
    - 子任务2: 实现核心业务逻辑（~200行，3个文件）
    - 子任务3: 实现API控制器和参数校验（~150行，2个文件）
    - 子任务4: 编写单元测试（~200行，3个文件）
```

---

## 注意事项

1. **预检必须完成**：不得跳过预检直接开始编码
2. **诚实预估**：预估应基于实际分析，不得刻意低估
3. **拆分粒度**：拆分后的子任务也应满足≤500行、≤10文件的限制
4. **复杂度评估**：边界条件和异常场景必须明确列出
