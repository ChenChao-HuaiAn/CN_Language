# 自检报告提示 (Self-Check Prompt)

## 使用时机

在代码交付之前，必须完成自检并生成自检报告。

## 自检指令

```
你的代码编写任务已完成。在交付之前，请完成以下自检：

## Step 1: 功能完整性检查

请确认：
- [ ] 所有方法都已完整实现，无 TODO/FIXME 占位
- [ ] 所有边界场景都已处理
- [ ] 所有异常情况都已捕获

## Step 2: 代码质量检查

请确认：
- [ ] 参数校验完备，无空指针风险
- [ ] 错误处理完整，有适当的日志记录
- [ ] 无硬编码魔法值
- [ ] 变量/方法命名清晰合理

## Step 3: 测试覆盖检查

请确认：
- [ ] 单元测试已编写
- [ ] 核心逻辑已覆盖
- [ ] 边界条件已测试
- [ ] 测试覆盖率 ≥ 80%

## Step 4: 规范一致性检查

请确认：
- [ ] 代码风格符合项目规范
- [ ] 复杂逻辑有注释说明
- [ ] API 接口符合项目约定
- [ ] 正确使用了项目公共组件

## Step 5: 生成自检报告

请按以下格式输出：

```markdown
## 代码交付自检报告

### 基本信息
- 任务名称：[任务名称]
- 涉及文件：[文件列表，每行一个]
- 代码行数：[总行数] 行
- 测试文件：[测试文件列表]

### 检查清单
- [ ] 方法全部实现，无TODO占位
- [ ] 参数校验完备
- [ ] 异常处理覆盖
- [ ] 无魔法值
- [ ] 单元测试通过
- [ ] 测试覆盖率达标
- [ ] 符合项目规范
- [ ] 复杂逻辑有注释说明

### 测试结果
- 单元测试：[X] 个用例，[X] 个通过
- 覆盖率：[X]%
- 失败用例：[如有失败，列出原因]

### 代码亮点
- [列出代码中的亮点或优化点]

### 待改进项（可选）
- [列出后续可优化的地方]

### 交付确认
**自检结果**：✅ 通过 / ❌ 不通过
**签名**：AI Code Quality Guard
**时间**：[YYYY-MM-DD HH:mm]
```
```

│  • 缺失的校验                    │

│  • 覆盖率不足                    │

              │
              ▼
┌─────────────────────────────────┐

              │
              ▼

│  3. 重新运行测试                 │

              ▼

```

## 注意事项

1. **诚实自检**：自检结果必须真实反映代码状态，不得隐瞒问题
2. **完整检查**：所有检查项都必须检查，不得跳过
3. **问题修复**：发现问题必须修复后才能交付
4. **签名确认**：自检通过后必须签名确认，承担质量责任
