# AI代码质量保障技能 (AI Code Quality Guard)

## 概述

本技能为AI Agent提供代码质量保障能力，确保每次代码交付都符合质量标准。基于四维度质量保障机制：

1. **预防机制** - 任务设计阶段的粒度控制和规范预检
2. **检测机制** - 代码编写阶段的禁区规则检测
3. **审批机制** - 代码交付前的自检报告
4. **反馈迭代** - 违规记录和经验沉淀

---

## 何时使用此技能 / When to Use

**使用场景：**
- 用户请求编写新代码或修改现有代码
- 功能实现任务开始前
- 代码重构任务开始前
- Bug修复任务开始前
- 任何涉及代码产出的任务

**不要使用：**
- 纯文档编写任务
- 项目规划或架构设计任务
- 仅阅读和分析代码的任务

## 核心规则

### 一、任务预检规则（Phase 1）

在开始编写代码之前，必须进行以下预检：

| 检查项 | 限制值 | 超限处理 |
|--------|--------|----------|
| **代码量预估** | ≤500行 | 超过则建议拆分为多个子任务 |
| **文件数预估** | ≤10个 | 超过则建议拆分为多个子任务 |
| **复杂度评估** | 明确输入/输出/边界 | 未明确则先澄清需求 |

#### 预检清单模板

```markdown
## 任务预检清单

### 任务信息
- 任务名称：[填写任务名称]
- 预估代码量：[X] 行（限制：500行）
- 预估文件数：[X] 个（限制：10个）

### 复杂度评估
- 输入定义：[明确输入数据/参数]
- 输出定义：[明确输出结果/返回值]
- 边界条件：[列出边界场景]

### 预检结果
- [ ] 代码量符合限制
- [ ] 文件数符合限制
- [ ] 复杂度已评估明确

**预检结论**：通过 / 需拆分
```

### 二、代码编写禁区（Phase 2）

以下行为严格禁止，检测到将自动驳回：

| 禁区规则 | 说明 | 检测方式 |
|----------|------|----------|
| **禁止TODO占位** | 方法内禁止写 `// TODO` 或 `// 待实现` 后直接返回 | 正则扫描 |
| **禁止空方法体** | 方法体不能为空或只抛出 `NotImplementedException` | AST分析 |
| **禁止省略校验** | 必须进行参数校验和边界处理 | 代码审查 |
| **禁止魔法值** | 禁止硬编码数字、字符串等魔法值 | 正则扫描 |

#### 禁区检测清单

```markdown
## 禁区检测清单

### 静态扫描项
- [ ] 无 TODO/FIXME/XXX 注释
- [ ] 无空方法体
- [ ] 无 NotImplementedException
- [ ] 无注释掉的代码块

### 代码质量项
- [ ] 参数校验完备
- [ ] 边界处理覆盖
- [ ] 无硬编码魔法值
- [ ] 错误处理完整

### 检测结果
**状态**：合格 / 不合格
**问题清单**：[列出发现的问题]
```

### 三、必须检查项（Phase 3）

代码编写完成后，必须进行以下检查：

| 检查项 | 要求 | 说明 |
|--------|------|------|
| **单元测试** | 覆盖核心逻辑 | 测试覆盖率 ≥80% |
| **错误处理** | 异常捕获和日志记录 | 关键路径必须有错误处理 |
| **代码风格** | 符合项目规范 | 遵循项目已有代码风格 |
| **注释说明** | 复杂逻辑必须有注释 | 解释"为什么"而非"是什么" |

### 四、自检报告格式（Phase 4）

代码交付前必须生成自检报告：

```markdown
## 代码交付自检报告

### 基本信息
- 任务名称：[任务名称]
- 涉及文件：[文件列表]
- 代码行数：[X] 行

### 检查清单
- [ ] 方法全部实现，无TODO占位
- [ ] 参数校验完备
- [ ] 异常处理覆盖
- [ ] 单元测试通过
- [ ] 符合项目规范
- [ ] 复杂逻辑有注释说明

### 测试结果
- 单元测试：[X] 个用例，[X] 个通过
- 覆盖率：[X]%

### 交付确认
**自检结果**：通过 / 不通过
**签名**：AI Code Quality Guard
**时间**：[YYYY-MM-DD HH:mm]
```

## 工作流程

### 完整执行流程

```
代码编写任务开始
       │
       ▼
┌─────────────────────────────────┐
│  Phase 1: 任务预检               │
│  • 代码量/文件数预估             │
│  • 复杂度评估                    │
│  • 超限则建议拆分                │
└─────────────┬───────────────────┘
              │ 通过
              ▼
┌─────────────────────────────────┐
│  Phase 2: 代码编写               │
│  • 加载禁区规则                  │
│  • 编写代码                      │
│  • 实时检测违规                  │
└─────────────┬───────────────────┘
              │ 编码完成
              ▼
┌─────────────────────────────────┐
│  Phase 3: 禁区检测               │
│  • TODO/空方法体扫描            │
│  • 魔法值检测                    │
│  • 参数校验检查                  │
│  ─────────────────────          │
│  有违规 → 自动修正 → 重新检测    │
└─────────────┬───────────────────┘
              │ 检测通过
              ▼
┌─────────────────────────────────┐
│  Phase 4: 自检报告               │
│  • 生成代码交付自检报告          │
│  • 确认所有检查项通过            │
│  ─────────────────────          │
│  未通过 → 补充完善 → 重新自检    │
└─────────────┬───────────────────┘
              │ 全部通过
              ▼
       代码交付完成
```
