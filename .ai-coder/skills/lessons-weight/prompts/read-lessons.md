# lessons.md 读取提示

## 强制指令

在读取 lessons.md 文件时，你**必须**遵循以下规则：

### 1. 优先读取顺序

```
第一步：读取 "⚠️ 高权重问题" 部分
        - 重点标记 🔴 权重 ≥ 15 的问题
        - 其次标记 🟡 权重 10-14 的问题

第二步：阅读这些问题并理解：
        - 问题类型和根本原因
        - 解决方案和预防措施
        - 对当前任务的潜在影响

第三步：在响应中确认已读取：
        "我已读取 lessons.md 中的 [X] 个高权重问题，将避免重复犯错"
```

### 2. 权重阈值行为

| 权重范围 | 行为要求 |
|---------|---------|
| ≥ 15 | **必须**在响应中提及，说明如何避免 |
| 10-14 | **应该**阅读，相关时提及 |
| 5-9 | 可选阅读 |
| < 5 | 可跳过 |

### 3. 任务类型匹配

| 当前任务类型 | 应查找的历史问题类型 |
|-------------|-------------------|
| 代码编写 | 逻辑错误、设计缺陷、任务未完成 |
| 配置修改 | 配置错误、集成问题 |
| 功能开发 | 设计缺陷、逻辑错误 |
| Bug 修复 | 逻辑错误、工具执行错误 |
| 集成开发 | 集成问题、设计缺陷 |
| 任务执行 | 任务未完成、工具执行错误 |
