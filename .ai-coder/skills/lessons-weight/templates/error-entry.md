# 错误记录模板

## 标准格式

```markdown
- [YYYY-MM-DD HH:mm] **问题类型**: [类型]
  - **描述**: [问题描述]
  - **原因**: [根本原因分析]
  - **解决**: [解决方案]
  - **预防**: [预防措施]
```

## 各类型示例

### 设计缺陷（基础权重: 10）

```markdown
- [2026-03-17 23:00] **问题类型**: 设计缺陷
  - **描述**: 记忆系统性能下降误报 - 阈值过低且无趋势检测
  - **原因**: warningThreshold=50MB/criticalThreshold=100MB对于功能丰富的VSCode扩展太低，且一超过阈值就触发降级，没有区分正常内存使用和内存泄漏
  - **解决**: 1.提高阈值到150MB/300MB 2.增加趋势检测功能analyzeTrend() 3.只有内存持续异常增长(每次>10MB)时才触发降级
  - **预防**: 设计阈值时应考虑实际使用场景，添加趋势检测避免误报
```

### 任务未完成（基础权重: 9）

```markdown
- [2026-03-17 23:00] **问题类型**: 任务未完成
  - **描述**: 记忆系统存储与内存优化方案阶段二未实施
  - **原因**: 阶段一基础设施已完成，但阶段二核心改造未执行，导致组件独立存在但未整合
  - **解决**: 重构MemoryStore使用适配器模式，实现延迟加载，添加自动迁移检测
  - **预防**: 每个阶段完成后进行集成验证，确保组件正确整合
```

### 逻辑错误（基础权重: 8）

```markdown
- [2026-03-17 23:00] **问题类型**: 逻辑错误
  - **描述**: AutoContextInjector调用错误方法导致上下文注入失效
  - **原因**: 错误调用addMemory()搜索方法而非addCustom()添加方法
  - **解决**: 修改为调用addCustom()方法直接添加格式化内容
  - **预防**: 调用方法前确认方法签名和预期行为
```

### 集成问题（基础权重: 7）

```markdown
- [2026-03-17 23:00] **问题类型**: 集成问题
  - **描述**: Wiki插件Webview与扩展通信失败
  - **原因**: 消息处理器缺少command类型处理
  - **解决**: 添加command消息类型处理逻辑
  - **预防**: 集成测试覆盖所有消息类型
```

### 配置错误（基础权重: 6）

```markdown
- [2026-03-17 23:00] **问题类型**: 配置错误
  - **描述**: CSP策略配置错误导致Webview空白
  - **原因**: script-src配置只允许nonce内联脚本，阻止外部JS加载
  - **解决**: 修改CSP策略允许从webview本地资源加载脚本
  - **预防**: 配置CSP策略时考虑所有资源加载需求
```

### 工具执行错误（基础权重: 2）

```markdown
- [2026-03-17 23:00] **问题类型**: 工具执行错误
  - **描述**: [RooCode#say] task xxx aborted
```

> 注意：工具执行错误通常为临时性问题，权重低，可压缩显示
