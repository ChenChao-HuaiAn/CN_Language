# 错误分析模板

此模板用于指导Agent对错误进行系统性分析，找出根本原因并制定修复方案。

## 使用说明

当发现错误时，使用此模板进行深入分析。分析应该全面、系统，并产生可执行的修复方案。

## 错误分析流程

### 第一步：收集错误信息

```markdown
## 错误信息收集

### 错误基本信息
- **错误类型**: [编译错误/运行时错误/逻辑错误/性能错误/安全错误]
- **发现时间**: {时间戳}
- **发现方式**: [用户报告/自动测试/代码审查/运行监控]
- **影响范围**: {描述影响范围}
- **严重程度**: [P0-紧急/P1-高/P2-中/P3-低]

### 错误详细描述
```
{粘贴完整的错误信息、堆栈跟踪、日志等}
```

### 复现步骤
1. {步骤1}
2. {步骤2}
3. {步骤3}
...

### 预期行为
{描述正确的行为应该是什么}

### 实际行为
{描述实际发生的行为}
```

### 第二步：定位错误位置

```markdown
## 错误定位

### 直接位置
- **文件**: {文件路径}
- **行号**: {行号范围}
- **代码片段**:
```{语言}
{相关代码片段}
```

### 相关代码
- **调用链**:
  1. {函数1} ({文件}:{行号})
  2. {函数2} ({文件}:{行号})
  3. {函数3} ({文件}:{行号})

- **依赖项**:
  - {依赖1}: {说明}
  - {依赖2}: {说明}

### 环境信息
- **操作系统**: {OS信息}
- **运行环境**: {运行时版本}
- **依赖版本**: {相关依赖版本}
- **配置**: {相关配置信息}
```

### 第三步：根因分析

使用"5个为什么"方法进行根因分析：

```markdown
## 根因分析（5个为什么）

### 为什么1：为什么会出现这个错误？
**回答**: {直接原因}
**证据**: {支持证据}

### 为什么2：为什么会出现{直接原因}？
**回答**: {更深层原因}
**证据**: {支持证据}

### 为什么3：为什么会出现{更深层原因}？
**回答**: {再深层原因}
**证据**: {支持证据}

### 为什么4：为什么会出现{再深层原因}？
**回答**: {更更深层原因}
**证据**: {支持证据}

### 为什么5：为什么会出现{更更深层原因}？
**回答**: {根本原因}
**证据**: {支持证据}

### 根本原因总结
{总结根本原因，这是需要解决的核心问题}
```

## 错误分析输出格式

最终输出应使用以下YAML格式：

```yaml
error_analysis:
  error_id: "{错误唯一标识}"
  timestamp: "{ISO 8601时间戳}"
  
  basic_info:
    type: "runtime_error"  # 编译错误/运行时错误/逻辑错误/性能错误/安全错误
    severity: "P1"  # P0-紧急/P1-高/P2-中/P3-低
    status: "analyzing"  # analyzing/fixing/fixed/closed
  
  description:
    expected: "预期行为"
    actual: "实际行为"
    impact: "影响描述"
  
  location:
    file: "文件路径"
    line_start: 10
    line_end: 20
    code_snippet: "代码片段"
  
  root_cause:
    direct_cause: "直接原因"
    deep_cause: "深层原因"
    root_cause: "根本原因"
    evidence: ["证据1", "证据2"]
  
  fix_plan:
    recommended_solution: "推荐方案"
    steps:
      - "步骤1"
      - "步骤2"
    estimated_time: "预计时间"
    rollback_plan: "回滚方案"
  
  prevention:
    short_term: ["短期措施1", "短期措施2"]
    mid_term: ["中期措施1"]
    long_term: ["长期措施1"]
  
  lessons:
    - insight: "洞察内容"
      context: "上下文"
      applicable_scenarios: "适用场景"
      confidence: 0.9
  
  collaboration_requests:
    - target_skill: "knowledge-accumulation"
      action: "store"
      data:
        category: "lesson"
        content: "教训内容"
```

## 注意事项

1. **不要假设**: 基于事实和证据进行分析，不要猜测
2. **深入根因**: 不要只修复表面问题，要找到根本原因
3. **全面考虑**: 考虑所有可能的影响和风险
4. **文档记录**: 详细记录分析过程，便于后续参考
5. **及时分享**: 将教训分享给团队，避免重复犯错
