# 反思提示模板

此模板用于指导Agent对已完成的任务进行系统性反思。

## 使用说明

在任务完成后，使用此模板进行反思。反思应该全面、客观，并产生可操作的改进建议。

## 反思流程

### 第一步：回顾任务

```markdown
## 任务回顾

### 原始需求
{记录用户的原始需求}

### 理解的需求
{记录Agent理解后的需求}

### 执行计划
{记录Agent制定的执行计划}
```

### 第二步：分析执行过程

```markdown
## 执行过程分析

### 采取的步骤
1. {步骤1}
2. {步骤2}
3. {步骤3}
...

### 每个步骤的结果
- 步骤1: {结果描述}
- 步骤2: {结果描述}
...

### 遇到的问题
- {问题1}: {如何解决的}
- {问题2}: {如何解决的}
...

### 做出的决策
- {决策1}: {原因和依据}
- {决策2}: {原因和依据}
...
```

### 第三步：评估输出质量

```markdown
## 输出质量评估

### 准确性评估
- [ ] 事实是否准确？
- [ ] 逻辑是否正确？
- [ ] 代码是否能运行？
- [ ] 是否有遗漏？

评分: [1-5分]
说明: {评分说明}

### 完整性评估
- [ ] 是否回答了所有问题？
- [ ] 是否覆盖了所有需求？
- [ ] 是否考虑了边界情况？
- [ ] 是否提供了必要的文档？

评分: [1-5分]
说明: {评分说明}

### 清晰性评估
- [ ] 表达是否清晰？
- [ ] 结构是否合理？
- [ ] 是否易于理解？
- [ ] 是否有歧义？

评分: [1-5分]
说明: {评分说明}

### 效率评估
- [ ] 是否采用了最优方法？
- [ ] 是否有不必要的步骤？
- [ ] 资源使用是否合理？
- [ ] 是否可以并行处理？

评分: [1-5分]
说明: {评分说明}
```

### 第四步：识别问题和改进点

```markdown
## 问题识别

### 高优先级问题
| 问题 | 类型 | 影响 | 根因 |
|------|------|------|------|
| {问题1} | {类型} | {影响} | {根因} |
| {问题2} | {类型} | {影响} | {根因} |

### 中优先级问题
| 问题 | 类型 | 影响 | 根因 |
|------|------|------|------|
| {问题1} | {类型} | {影响} | {根因} |

### 低优先级问题
| 问题 | 类型 | 影响 | 根因 |
|------|------|------|------|
| {问题1} | {类型} | {影响} | {根因} |

问题类型说明：
- error: 错误，需要立即修复
- warning: 警告，应该修复
- improvement: 改进建议，可选修复
```

### 第五步：生成改进建议

```markdown
## 改进建议

### 立即改进（高优先级）
1. **{改进项1}**
   - 问题: {关联的问题}
   - 行动: {具体行动}
   - 预期效果: {预期效果}

### 短期改进（中优先级）
1. **{改进项1}**
   - 问题: {关联的问题}
   - 行动: {具体行动}
   - 预期效果: {预期效果}

### 长期改进（低优先级）
1. **{改进项1}**
   - 问题: {关联的问题}
   - 行动: {具体行动}
   - 预期效果: {预期效果}
```

- **适用场景**: {这个教训适用于哪些场景}
- **关联知识**: {相关的知识点或技能}
- **置信度**: [0-1]
```

```

## 反思输出格式

最终输出应使用以下YAML格式：

```yaml
reflection_result:
  task_id: "{任务唯一标识}"
  timestamp: "{ISO 8601时间戳}"
  
  summary:
    overall_score: 4  # 总体评分 1-5
    main_achievements:  # 主要成就
      - "{成就1}"
      - "{成就2}"
    main_issues:  # 主要问题
      - "{问题1}"
      - "{问题2}"
  
  issues:
    - type: "error"  # error|warning|improvement
      severity: "high"  # high|medium|low
      description: "问题描述"
      location: "问题位置"
      root_cause: "根本原因"
      fix_suggestion: "修复建议"
  
  improvements:
    - category: "knowledge"  # knowledge|capability|performance
      action: "具体行动"
      priority: "high"  # high|medium|low
      related_skill: "关联技能"  # 可选
  
  lessons_learned:
    - insight: "洞察内容"
      context: "上下文"
      applicable_scenarios: "适用场景"
      confidence: 0.8  # 置信度 0-1
  
  collaboration_requests:  # 协作请求
    - target_skill: "knowledge-accumulation"
      action: "store"
      data: {请求数据}
```

## 注意事项

1. **保持客观**: 不要为错误找借口，客观分析问题
2. **深入根因**: 不要只停留在表面，追问"为什么"
3. **可操作性**: 改进建议要具体、可执行
4. **及时记录**: 教训要及时记录，避免遗忘
5. **持续改进**: 反思是一个持续的过程，不是一次性的
