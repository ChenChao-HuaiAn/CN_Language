# CN_Language - 中文编程语言项目

## 项目概述

CN_Language是一个为中文用户设计的编程语言项目，旨在提供友好、直观的编程体验。项目采用分层架构和SOLID设计原则，确保代码的可维护性、可扩展性和可测试性。

## 架构设计

### 三层分层架构

项目采用清晰的三层架构设计：

```
┌─────────────────────────────────────────┐
│           应用层 (Application Layer)    │
│  - 命令行界面 (CLI)                     │
│  - REPL交互环境                         │
│  - 调试器界面                           │
│  - 集成开发环境插件                     │
├─────────────────────────────────────────┤
│           核心层 (Core Layer)           │
│  - 编译器前端 (词法分析、语法分析)      │
│  - 编译器后端 (代码生成、优化)          │
│  - 运行时系统 (内存管理、执行引擎)      │
│  - 标准库实现                           │
├─────────────────────────────────────────┤
│       基础设施层 (Infrastructure Layer) │
│  - 基础数据结构 (字符串、数组、哈希表)  │
│  - 内存管理 (分配器、垃圾回收)          │
│  - 错误处理 (异常、日志)                │
│  - 工具库 (UTF-8处理、文件IO、网络)     │
│  - 平台抽象层 (操作系统接口)            │
└─────────────────────────────────────────┘
```

### 依赖规则

1. **单向依赖**：上层可以依赖下层，下层不能依赖上层
2. **接口依赖**：层间通过抽象接口通信，而非具体实现
3. **依赖注入**：通过依赖注入管理层间依赖关系

## 构建说明

### 环境要求

- GCC编译器（支持C99）
- Make构建工具
- （可选）clang-format（代码格式化）
- （可选）cppcheck（静态分析）

### 构建命令

```bash
# 构建发布版本
make

# 构建调试版本
make BUILD_CONFIG=debug

# 构建并运行测试
make test

# 运行调试器
make debug

# 清理构建产物
make clean

# 显示帮助信息
make help
```

- `make all` - 构建所有目标（默认）
- `make clean` - 清理所有构建产物
- `make test` - 构建并运行测试
- `make debug` - 构建并运行调试器
- `make debug-build` - 构建调试版本
- `make deps-graph` - 生成依赖关系图（需要graphviz）
- `make stats` - 显示代码统计信息
- `make lint` - 运行代码格式检查
- `make format` - 格式化代码
- `make analyze` - 运行静态分析

## 开发规范

### 代码规范

1. **命名规范**：
   - 结构体：`Stru_`前缀，`_t`后缀（如`Stru_Token_t`）
   - 枚举：`Eum_`前缀（如`Eum_TokenType`）
   - 函数：`F_`前缀（如`F_token_create`）
   - 常量：`CN_`前缀，全大写（如`CN_MAX_TOKEN_LENGTH`）

2. **文件规范**：
   - 每个.c文件不超过500行
   - 每个函数不超过50行
   - 头文件包含防止重复包含的保护

3. **注释规范**：
   - 文件头注释：包含文件名、功能、作者、日期、版权
   - 函数注释：使用Doxygen格式（@brief、@param、@return）
   - 复杂逻辑注释：解释算法和设计思路

### 架构原则

1. **单一职责原则**：每个模块只负责一个功能领域
2. **开闭原则**：通过接口支持扩展，对修改封闭
3. **里氏替换原则**：接口实现必须完全遵守契约
4. **接口隔离原则**：为不同客户端提供专用接口
5. **依赖倒置原则**：高层模块定义接口，低层模块实现接口

## 模块说明

### 基础设施层

#### 内存管理模块（CN_memory）
提供统一的内存管理接口，支持多种分配策略：
- 系统分配器（malloc/free）
- 对象池分配器
- 区域分配器
- 调试分配器（检测内存错误）

#### 容器模块（CN_array）
提供动态数组容器，支持：
- 自动扩容/缩容
- 多种元素类型
- 迭代器支持
- 排序和搜索

#### 字符串工具模块（CN_string）
提供UTF-8字符串处理，支持：
- 多编码转换
- 字符串操作（拼接、分割、替换）
- 格式化输出
- 国际化支持

### 核心层

#### 词法分析模块（lexer）
- Token定义和分类
- 关键字识别
- 词法错误处理

#### 类型系统模块（types）
- 基本类型定义（整数、小数、字符串、布尔）
- 复合类型（数组、结构体、枚举）
- 类型检查和转换
- 引用计数内存管理

### 应用层

#### 调试器模块（debugger）
- 调试信息输出
- 断点管理
- 变量查看
- 调用栈跟踪
