## 1. 问题分析

### 1.1 当前状态

通过分析 [`examples/module-system/综合测试.cn`](examples/module-system/综合测试.cn:1) 和生成的 [`examples/module-system/综合测试.c`](examples/module-system/综合测试.c:1)，发现以下问题：

#### 问题1：前向声明来源不明确

**生成的C代码**（第9-15行）：
```c
// Forward Declarations - 从导入模块
long long 加法(long long a, long long b);
long long 减法(long long a, long long b);
long long 乘法(long long a, long long b);
long long 测试函数();
long long 获取版本();
```

**问题**：这些前向声明是硬编码生成的，不是从导入模块的元数据中动态获取的。

#### 问题2：单文件编译模式限制

当前编译器主要支持单文件编译模式：
- [`src/cli/cnc/main.c`](src/cli/cnc/main.c:1) 中的编译流程只处理单个源文件
- 导入的外部模块只进行符号解析，不生成对应的C代码

#### 问题3：模块间依赖未正确处理

从 [`src/backend/cgen/module_cgen.c`](src/backend/cgen/module_cgen.c:1) 可以看到：
- 已有 `cn_cgen_module_header()` 和 `cn_cgen_module_impl()` 函数
- 但这些函数在单文件编译模式下未被调用
- 多文件编译流程未完整实现

### 2.2 核心改进点

#### 改进1：增强前向声明生成

**目标**：从导入模块的元数据中动态获取函数签名，生成正确的前向声明。

**实现位置**：[`src/backend/cgen/cgen.c`](src/backend/cgen/cgen.c:1382) 的前向声明生成部分

**当前代码**：
```c
fprintf(file, "// Forward Declarations\n");
CnIrFunction *func = module->first_func;
while (func) {
    // 只生成当前模块的函数声明
    ...
}
```

**改进方案**：
```c
fprintf(file, "// Forward Declarations - 从导入模块\n");

// 1. 遍历导入语句，获取导入的模块
for (size_t i = 0; i < program->import_count; i++) {
    CnAstImport *import = program->imports[i];
    
    // 2. 从模块加载器获取模块元数据
    CnModuleMetadata *metadata = cn_module_loader_load(loader, import->module_id);
    if (!metadata) continue;
    
    // 3. 遍历导出符号，生成前向声明
    for (size_t j = 0; j < metadata->export_count; j++) {
        CnExportedSymbol *symbol = &metadata->exports[j];
        
        // 4. 检查是否为选择性导入
        if (import->is_selective) {
            int found = 0;
            for (size_t k = 0; k < import->symbol_count; k++) {
                if (strcmp(import->symbols[k], symbol->name) == 0) {
                    found = 1;
                    break;
                }
            }
            if (!found) continue;
        }
        
        // 5. 生成前向声明
        if (symbol->is_function) {
            CnSemSymbol *sym_info = (CnSemSymbol *)symbol->symbol_info;
            fprintf(file, "%s %s(%s);\n", 
                    get_c_type_str(sym_info->type->as.function.return_type),
                    symbol->name,
                    format_params(sym_info->type));
        }
    }
}
```

#### 改进2：实现多文件编译模式

**目标**：为每个导入的模块生成独立的 `.h` 和 `.c` 文件。

**新增命令行选项**：
```
cnc --multi-file 综合测试.cn -o build/
```

**实现流程**：

```c
// 在 src/cli/cnc/main.c 中添加多文件编译模式
int compile_multi_file(const char *source_path, const char *output_dir) {
    // 1. 创建模块加载器
    CnModuleLoader *loader = cn_module_loader_create();
    
    // 2. 解析主模块
    CnAstProgram *program = parse_source(source_path);
    
    // 3. 构建模块依赖图
    cn_sem_build_scopes_with_loader(program, diagnostics, loader, source_path);
    
    // 4. 获取模块初始化顺序
    CnModuleId **init_order;
    size_t init_count;
    cn_module_loader_get_init_order(loader, &init_order, &init_count);
    
    // 5. 创建多文件编译上下文
    CnMultiFileCompileContext *ctx = cn_multi_compile_ctx_create(output_dir, loader);
    
    // 6. 为每个模块添加编译单元
    for (size_t i = 0; i < init_count; i++) {
        CnModuleMetadata *meta = cn_module_loader_load(loader, init_order[i]);
        cn_multi_compile_ctx_add_unit(ctx, init_order[i], meta->file_path, 
                                       i == init_count - 1);  // 最后一个是入口模块
    }
    
    // 7. 生成所有模块的代码
    cn_cgen_all_modules(ctx);
    
    // 8. 清理
    cn_multi_compile_ctx_free(ctx);
    cn_module_loader_free(loader);
}
```

## 5. 代码示例

### 5.1 改进后的前向声明生成

```c
// src/backend/cgen/import_cgen.c（新文件）

#include "cnlang/backend/cgen.h"
#include "cnlang/frontend/module_loader.h"
#include "cnlang/frontend/semantics.h"
#include <stdio.h>
#include <string.h>

/**
 * @brief 生成导入模块的前向声明
 * 
 * @param ctx 代码生成上下文
 * @param program AST程序节点
 * @param loader 模块加载器
 */
void cn_cgen_import_declarations(CnCCodeGenContext *ctx, 
                                  CnAstProgram *program,
                                  CnModuleLoader *loader) {
    if (!ctx || !ctx->output_file || !program || !loader) {
        return;
    }
    
    FILE *f = ctx->output_file;
    
    fprintf(f, "// Forward Declarations - 从导入模块\n");
    
    // 遍历所有导入语句
    for (size_t i = 0; i < program->import_count; i++) {
        CnAstImport *import = program->imports[i];
        if (!import->module_path) continue;
        
        // 构建模块ID
        CnModuleId *module_id = cn_module_id_create(import->module_path);
        if (!module_id) continue;
        
        // 加载模块元数据
        CnModuleMetadata *metadata = cn_module_loader_load(loader, module_id);
        if (!metadata) {
            cn_module_id_free(module_id);
            continue;
        }
        
        // 遍历导出符号
        for (size_t j = 0; j < metadata->export_count; j++) {
            CnExportedSymbol *symbol = &metadata->exports[j];
            
            // 选择性导入检查
            if (import->is_selective && import->symbols) {
                int found = 0;
                for (size_t k = 0; k < import->symbol_count; k++) {
                    if (strcmp(import->symbols[k], symbol->name) == 0) {
                        found = 1;
                        break;
                    }
                }
                if (!found) continue;
            }
            
            // 生成函数前向声明
            if (symbol->is_function && symbol->symbol_info) {
                CnSemSymbol *sym = (CnSemSymbol *)symbol->symbol_info;
                cn_cgen_function_declaration(f, symbol->name, sym->type);
            }
        }
        
        cn_module_id_free(module_id);
    }
    
    fprintf(f, "\n");
}

/**
 * @brief 生成函数声明
 */
void cn_cgen_function_declaration(FILE *f, const char *name, CnType *func_type) {
    if (!f || !name || !func_type || func_type->kind != CN_TYPE_FUNCTION) {
        return;
    }
    
    // 返回类型
    const char *ret_type = get_c_type_str(func_type->as.function.return_type);
    fprintf(f, "%s %s(", ret_type, name);
    
    // 参数列表
    if (func_type->as.function.param_count == 0) {
        fprintf(f, "void");
    } else {
        for (size_t i = 0; i < func_type->as.function.param_count; i++) {
            if (i > 0) fprintf(f, ", ");
            const char *param_type = get_c_type_str(func_type->as.function.param_types[i]);
            fprintf(f, "%s arg%zu", param_type, i);
        }
    }
    
    fprintf(f, ");\n");
}
```

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 模块元数据不完整 | 前向声明生成失败 | 在模块加载时验证导出符号信息 |
| 循环导入 | 编译死锁 | 使用模块加载器的循环检测功能 |
| 符号名冲突 | 链接错误 | 使用限定符号名（模块名__符号名） |
| 增量编译不一致 | 代码生成错误 | 使用文件修改时间戳检测变更 |

## 8. 总结

本方案通过以下改进解决模块系统代码生成问题：

1. **增强前向声明生成**：从模块元数据动态获取函数签名
2. **实现多文件编译模式**：为每个模块生成独立的 `.h` 和 `.c` 文件
3. **完善模块头文件生成**：确保导出符号声明正确
4. **添加导入模块头文件包含**：在主文件中包含所有依赖模块的头文件

这些改进将使CN语言的模块系统更加完善，支持真正的多文件编译和模块化开发。
