## 一、设计概述

### 1.1 设计目标

为CN语言引入C++风格的面向对象编程支持，包括：
- 类定义与对象实例化
- 多继承支持
- 虚函数与多态
- 访问控制（公开/私有/保护）
- 构造函数与析构函数
- 抽象类与纯虚函数

### 1.2 与现有系统的关系

| 现有特性 | OOP引入后的处理 |
|---------|----------------|
| `结构体` | 保持不变，作为纯数据聚合类型 |
| `枚举` | 保持不变 |
| 模块系统 | 类可在模块内定义，支持导出 |
| 函数 | 保持独立函数，同时支持成员函数 |

### 1.3 预留关键字激活

以下预留关键字将被激活使用：

| 关键字 | 用途 |
|--------|------|
| `类` | 类定义 |
| `接口` | 接口定义（纯抽象类） |
| `保护` | 保护访问级别 |
| `虚拟` | 虚函数声明 |
| `重写` | 重写虚函数标记 |
| `抽象` | 抽象类/纯虚函数标记 |

### 1.4 符号语法设计

不新增关键字，使用符号和现有语法结构实现OOP功能：

| 功能 | 语法设计 | 说明 |
|------|---------|------|
| 继承声明 | `类 派生类: 基类` | 使用冒号，类似C++/Python |
| 多继承 | `类 派生类: 基类1, 基类2` | 逗号分隔 |
| 构造函数 | `函数 类名()` | 与类同名的函数 |
| 析构函数 | `函数 ~类名()` | 波浪号前缀 |
| 父类访问 | `基类.成员` | 使用现有关键字组合 |
| this指针 | `自身.成员` | 使用现有关键字组合 |

## 二、语法设计

### 2.1 类定义基本语法

```cn
类 类名 {
    访问标签:
        成员声明
        成员声明
        ...
};
```

### 2.2 访问控制

```cn
类 示例类 {
公开:                          // public
    整数 公开变量;
    函数 公开方法() { ... }

私有:                          // private
    整数 私有变量;
    函数 私有方法() { ... }

保护:                          // protected
    整数 保护变量;
    函数 保护方法() { ... }
};
```

**默认访问级别**：
- 类成员默认为 `私有`
- 结构体成员默认为 `公开`（保持现有行为）

### 2.3 继承语法

#### 单继承

```cn
类 派生类: 基类 {
    // 类体
};
```

#### 多继承

```cn
类 派生类: 基类1, 基类2, 基类3 {
    // 类体
};
```

#### 继承访问控制

```cn
类 派生类: 公开 基类 {        // public继承（默认）
    // ...
};

类 派生类: 私有 基类 {        // private继承
    // ...
};

类 派生类: 保护 基类 {        // protected继承
    // ...
};
```

### 2.4 虚函数与多态

**成员函数返回值类型语法**：
- 无返回值：`函数 函数名(参数) { ... }` 或 `函数 函数名(参数) -> 空类型 { ... }`
- 有返回值：`函数 函数名(参数) -> 返回类型 { ... }`

**虚函数与返回值类型的组合**：

| 情况 | 语法 |
|------|------|
| 无返回值虚函数 | `虚拟 函数 函数名() { ... }` |
| 无返回值纯虚函数 | `虚拟 函数 函数名() 抽象;` |
| 无返回值重写函数 | `函数 函数名() 重写 { ... }` |
| 有返回值虚函数 | `虚拟 函数 函数名() -> 类型 { ... }` |
| 有返回值纯虚函数 | `虚拟 函数 函数名() -> 类型 抽象;` |
| 有返回值重写函数 | `函数 函数名() -> 类型 重写 { ... }` |

```cn
类 动物 {
公开:
    // 无返回值虚函数
    虚拟 函数 说话() {
        打印("动物发出声音");
    }
    
    // 无返回值纯虚函数
    虚拟 函数 移动() 抽象;
    
    // 有返回值虚函数
    虚拟 函数 获取名字() -> 字符串 {
        返回 "动物";
    }
    
    // 有返回值纯虚函数
    虚拟 函数 获取年龄() -> 整数 抽象;
};

类 狗: 动物 {
公开:
    // 无返回值重写
    函数 说话() 重写 {
        打印("汪汪汪");
    }
    
    // 无返回值重写（实现纯虚函数）
    函数 移动() 重写 {
        打印("四条腿跑");
    }
    
    // 有返回值重写
    函数 获取名字() -> 字符串 重写 {
        返回 "狗";
    }
    
    // 有返回值重写（实现纯虚函数）
    函数 获取年龄() -> 整数 重写 {
        返回 3;
    }
};
```

### 2.5 成员函数返回值类型

CN语言的成员函数支持两种返回值声明方式：

**语法规则**：
```
成员函数 ::= "函数" 函数名 "(" 参数列表? ")" ("->" 返回类型)? 函数体
```

**两种形式**：

| 形式 | 语法 | 说明 |
|------|------|------|
| 无返回值 | `函数 函数名(参数) { ... }` | 不需要返回值，可省略 `-> 空类型` |
| 有返回值 | `函数 函数名(参数) -> 返回类型 { ... }` | 必须使用 `返回` 语句返回指定类型的值 |

**示例**：
```cn
类 示例类 {
公开:
    // 无返回值函数（两种写法等价）
    函数 打印信息() {
        打印("无返回值");
    }
    
    函数 打印信息2() -> 空类型 {  // 显式声明空类型
        打印("无返回值");
    }
    
    // 有返回值函数
    函数 获取值() -> 整数 {
        返回 42;
    }
    
    函数 获取名字() -> 字符串 {
        返回 "示例";
    }
    
    函数 计算(整数 a, 整数 b) -> 整数 {
        返回 a + b;
    }
};
```

**重写时的返回值类型**：
- 重写函数必须与基类虚函数的返回值类型一致
- 协变返回类型（返回派生类指针/引用）暂不支持

```cn
类 基类 {
公开:
    虚拟 函数 获取值() -> 整数 抽象;
    虚拟 函数 执行() 抽象;  // 无返回值
};

类 派生类: 基类 {
公开:
    函数 获取值() -> 整数 重写 {  // 必须保持返回类型一致
        返回 100;
    }
    
    函数 执行() 重写 {  // 无返回值
        打印("执行中");
    }
};
```

### 2.6 构造函数与析构函数

构造函数使用与类同名的函数，析构函数使用 `~类名`：

```cn
类 学生 {
公开:
    字符串 姓名;
    整数 年龄;
    
    // 默认构造函数（与类同名）
    函数 学生() {
        姓名 = "未知";
        年龄 = 0;
    }
    
    // 带参数构造函数
    函数 学生(字符串 名, 整数 龄) {
        姓名 = 名;
        年龄 = 龄;
    }
    
    // 析构函数（波浪号前缀）
    函数 ~学生() {
        打印("学生对象被销毁");
    }
};
```

### 2.6 构造函数成员初始化

构造函数支持两种初始化成员变量的方式：

**方式一：初始化列表**（推荐用于常量成员和引用成员）

```cn
类 矩形 {
公开:
    整数 宽度;
    整数 高度;
    
    函数 矩形(整数 宽, 整数 高) : 宽度(宽), 高度(高) {
        打印("矩形创建完成");
    }
};
```

**方式二：函数体内部赋值**

```cn
类 矩形 {
公开:
    整数 宽度;
    整数 高度;
    
    函数 矩形(整数 宽, 整数 高) {
        宽度 = 宽;
        高度 = 高;
        打印("矩形创建完成");
    }
};
```

**两种方式的区别**：

| 特性 | 初始化列表 | 函数体赋值 |
|------|-----------|-----------|
| 执行时机 | 对象构造时 | 对象构造后 |
| 常量成员 | ✅ 支持 | ❌ 不支持 |
| 引用成员 | ✅ 支持 | ❌ 不支持 |
| 无默认构造的成员对象 | ✅ 支持 | ❌ 不支持 |
| 普通成员 | ✅ 支持 | ✅ 支持 |
| 性能 | 略优（直接初始化） | 略差（先默认构造再赋值） |

**混合使用**：

```cn
类 学生 {
公开:
    字符串 姓名;
    整数 年龄;
    常量 整数 学号;  // 常量成员必须用初始化列表
    
    函数 学生(字符串 名, 整数 龄, 整数 号) : 学号(号) {
        姓名 = 名;           // 函数体赋值
        年龄 = 龄;           // 函数体赋值
        // 学号 = 号;        // 错误！常量不能在函数体赋值
    }
};
```

### 2.7 父类构造函数调用

使用 `基类` 关键字调用父类构造函数：

```cn
类 动物 {
保护:
    字符串 名字;
    
公开:
    函数 动物(字符串 名) : 名字(名) {}
};

类 狗: 动物 {
公开:
    函数 狗(字符串 名) : 基类(名) {
        打印("狗创建完成");
    }
};
```

### 2.8 接口定义

接口使用 `接口` 关键字定义，本质上是纯抽象类：

```cn
接口 可序列化 {
    虚拟 函数 序列化() -> 字符串 抽象;
    虚拟 函数 反序列化(字符串 数据) -> 空类型 抽象;
};

类 用户: 可序列化 {
公开:
    字符串 姓名;
    
    函数 序列化() -> 字符串 重写 {
        返回 姓名;
    }
    
    函数 反序列化(字符串 数据) -> 空类型 重写 {
        姓名 = 数据;
    }
};
```

## 三、核心特性

### 3.1 特性矩阵

| 特性 | 支持程度 | 说明 |
|------|---------|------|
| 类定义 | ✅ 完整支持 | 包含数据成员和成员函数 |
| 多继承 | ✅ 完整支持 | 需处理菱形继承问题 |
| 虚函数 | ✅ 完整支持 | 支持运行时多态 |
| 纯虚函数 | ✅ 完整支持 | 定义抽象类 |
| 访问控制 | ✅ 完整支持 | 公开/私有/保护 |
| 构造函数 | ✅ 完整支持 | 支持重载和初始化列表 |
| 析构函数 | ✅ 完整支持 | 支持虚析构函数 |
| 接口 | ✅ 完整支持 | 作为纯抽象类的语法糖 |
| 静态成员 | ✅ 完整支持 | 静态变量和静态方法 |
| 友元 | ⏳ 暂不支持 | 可作为后续扩展 |
| 运算符重载 | ⏳ 暂不支持 | 可作为后续扩展 |

### 3.2 菱形继承处理

采用C++的**虚继承**机制解决菱形继承问题：

```cn
类 动物 {
公开:
    整数 年龄;
};

// 虚继承基类（使用 虚拟 关键字修饰继承）
类 飞行动物: 虚拟 动物 {
    // ...
};

类 游泳动物: 虚拟 动物 {
    // ...
};

// 鸭子同时继承飞行动物和游泳动物，但只有一个动物基类子对象
类 鸭子: 飞行动物, 游泳动物 {
    // 年龄变量只有一份，不会产生二义性
};
```

**语法说明**：使用 `虚拟` 关键字修饰继承，表示虚继承

### 3.3 this指针

在成员函数中自动提供 `自身` 关键字（对应C++的this）：

```cn
类 计数器 {
公开:
    整数 值;
    
    函数 增加() {
        自身.值 = 自身.值 + 1;    // 显式使用自身
    }
    
    函数 获取值() -> 整数 {
        返回 值;                   // 隐式使用成员
    }
};
```

**说明**：`自身` 是现有预留关键字，用于引用当前对象实例

## 五、代码生成策略

### 5.1 类到C的映射

CN语言的类将编译为C结构体 + 函数指针 + 虚函数表：

**CN代码**：
```cn
类 动物 {
公开:
    整数 年龄;
    
    虚拟 函数 说话();
};

类 狗: 动物 {
公开:
    函数 说话() 重写 {
        打印("汪汪汪");
    }
};
```

**生成的C代码**：
```c
// 虚函数表
typedef struct AnimalVTable {
    void (*speak)(void* self);
} AnimalVTable;

// 基类结构体
typedef struct Animal {
    AnimalVTable* vtable;
    long long age;
} Animal;

// 派生类结构体
typedef struct Dog {
    Animal base;  // 基类子对象
    // Dog特有成员...
} Dog;

// 虚函数表实例
AnimalVTable g_dog_vtable = {
    .speak = Dog_speak
};

// 构造函数
void Dog_constructor(Dog* self) {
    self->base.vtable = &g_dog_vtable;
    self->base.age = 0;
}

// 成员函数实现
void Dog_speak(void* self) {
    cn_rt_print_string("汪汪汪");
}
```

### 5.2 多继承对象布局

```cn
类 A {
    整数 a;
};

类 B {
    整数 b;
};

类 C: A, B {
    整数 c;
};
```

**生成的C代码**：
```c
typedef struct A {
    long long a;
} A;

typedef struct B {
    long long b;
} B;

typedef struct C {
    A a_base;      // 第一个基类
    B b_base;      // 第二个基类
    long long c;   // C的成员
} C;
```

### 5.3 虚继承对象布局

对于虚继承，使用虚基类指针：

```c
typedef struct FlyingAnimal {
    void* vbptr;        // 虚基类指针
    // FlyingAnimal特有成员...
} FlyingAnimal;

typedef struct Duck {
    FlyingAnimal fa_base;
    SwimmingAnimal sa_base;
    Animal* virtual_base;  // 指向共享的Animal子对象
} Duck;
```

            返回 "文件内容";
        }

            打印("向文件" + 文件路径 + "写入: " + 数据);
        }
    };
}

## 九、设计决策总结

### 9.1 已确定的语法设计

| 功能 | 语法 | 说明 |
|------|------|------|
| 继承声明 | `类 派生类: 基类` | 使用冒号，类似C++/Python |
| 多继承 | `类 派生类: 基类1, 基类2` | 逗号分隔 |
| 虚继承 | `类 派生类: 虚拟 基类` | `虚拟` 关键字修饰基类 |
| 构造函数 | `函数 类名()` | 与类同名的函数 |
| 析构函数 | `函数 ~类名()` | 波浪号前缀 |
| 父类访问 | `基类.成员` 或 `基类(参数)` | 使用 `基类` 关键字 |
| this指针 | `自身.成员` | 使用 `自身` 关键字 |

### 9.2 关键字使用情况

**激活的预留关键字**：
- `类` - 类定义
- `接口` - 接口定义
- `保护` - 保护访问级别
- `虚拟` - 虚函数声明、虚继承修饰
- `重写` - 重写虚函数标记
- `抽象` - 抽象类/纯虚函数标记

**需要新增的关键字**：
- `自身` - this指针，引用当前对象实例
- `基类` - 访问父类成员或调用父类构造函数

**符号语法**：
- `:` - 继承声明分隔符
- `~` - 析构函数前缀

### 9.3 实现优先级

建议按以下顺序实现：
1. 类定义基础 + 成员函数
2. 构造函数/析构函数
3. 单继承 + 虚函数
4. 多继承
5. 虚继承
6. 接口

### 9.4 待后续讨论

1. **模块导出**：类如何与 `公开`/`私有` 模块可见性交互？
2. **函数指针**：成员函数指针如何支持？
3. **模板**：类模板如何设计（作为后续泛型支持的一部分）？

**虚基类共享机制**：
- 虚基类子对象放在对象布局的最后
- 通过vbptr（虚基类指针）访问虚基类成员

**构造函数委托**：
- 虚基类由最派生类直接构造
- 非虚基类按继承顺序构造
- 析构顺序与构造相反
