### 五、错误类型分析

#### 5.1 🔴 高优先级：CN语言不支持联合体（union）语法

**错误特征**：
```
[DEBUG PARSER] 无法识别的token: kind=80, line=36, col=21
语法错误：缺少预期的标记
```

**问题文件**：
- `src_cn/诊断/诊断恢复.cn` - 使用 `联合体 参数值`
- `src_cn/诊断/诊断消息表.cn` - 使用 `联合体 参数值`
- `src_cn/诊断/诊断修复.cn` - 使用 `联合体 参数值`

**根本原因**：
CN语言语法规范中没有定义 `联合体` 关键字。根据 [`plans/001 CN Language语法规范设计文档.md`](plans/001 CN Language语法规范设计文档.md) 第1.2节，CN语言只支持以下类型关键字：
- `整数`、`小数`、`字符串`、`布尔`、`空类型`、`结构体`、`枚举`

**修复建议**：
将 `联合体` 改为 `结构体`，或使用多个独立字段替代联合体。

---

```
语义错误：未定义的标识符 '当前词元'
```

**根本原因**：

3. 导出的符号未正确添加到当前作用域

#### 5.3 🟡 低优先级：类型不匹配错误

**错误特征**：
```
语义错误：函数 '追加整数' 参数 1 类型不匹配 (arg_kind=20, param_kind=1)
```

**影响范围**：
- 代码生成器模块：2个文件
- IR生成器模块：部分文件

**根本原因**：
`arg_kind=20` 表示 `CN_TYPE_UNKNOWN`（未知类型），类型系统在参数传递时未能正确处理未知类型。

**修复方案A**：改为结构体
```cn

    整数 标记值            // Token类型值
}
```

**修复方案B**：使用类型标签+独立字段
```cn
结构体 参数值 {
    整数 类型标签          // 0=字符串, 1=整数, 2=标记
    常量字符串 字符串值    // 字符串值

}
```
