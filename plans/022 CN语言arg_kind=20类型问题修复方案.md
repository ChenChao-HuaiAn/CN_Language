# CN语言 arg_kind=20 类型问题修复方案

**创建时间**: 2026-04-10 16:07

---

## 一、问题定义

### 1.1 问题现象

编译 src_cn 项目时，多个文件出现以下错误：

```
语义错误：函数 '追加整数' 参数 1 类型不匹配 (arg_kind=20, param_kind=1)
语义错误：函数 '创建局部变量映射' 参数 2 类型不匹配 (arg_kind=20, param_kind=5)
```

### 1.2 问题根源

**arg_kind=20 对应 `CN_TYPE_UNKNOWN`（未知类型）**

根据 [`include/cnlang/frontend/semantics.h:48`](include/cnlang/frontend/semantics.h:48) 的定义：

```c
typedef enum CnTypeKind {
    // ... 其他类型 ...
    CN_TYPE_UNKNOWN = 20  // 未知类型
} CnTypeKind;
```

### 1.3 问题分析

1. **模块导入时类型信息丢失**
   - 函数参数类型在跨模块导入时未完整解析
   - 类型被错误地标记为 `CN_TYPE_UNKNOWN`

2. **类型兼容性检查的限制**
   - 根据 [`src/semantics/symbols/type_system.c:275-278`](src/semantics/symbols/type_system.c:275)：
   ```c
   // UNKNOWN 类型可以接受任何类型（用于内置函数如 类型大小）
   if (b->kind == CN_TYPE_UNKNOWN) {
       return true;
   }
   ```
   - 只允许 `CN_TYPE_UNKNOWN` 作为**形式参数类型**（b），不允许作为**实际参数类型**（a）

## 三、修复方案

### 方案A：增强模块导入时的类型解析（高优先级）

**目标**：确保模块导入时函数参数类型完整解析

**修改文件**：

#### 1. `src/frontend/module_loader.c` - 模块加载器

**修改内容**：
- 在加载模块时，确保所有导出符号的类型信息完整
- 添加类型解析的延迟处理机制

**代码位置**：模块符号导出逻辑

#### 2. `src/semantics/resolution/scope_builder.c` - 作用域构建器

**修改内容**：
- 在 `cn_sem_build_scopes` 函数中，处理导入符号时完整解析类型
- 添加类型信息的传递逻辑

**关键函数**：
- `cn_sem_build_scopes()` - 主入口
- 导入语句处理逻辑

#### 3. `src/semantics/checker/semantic_passes.c` - 语义检查

**修改内容**：
- 在类型推断时，如果遇到 `CN_TYPE_UNKNOWN`，尝试从符号表中获取完整类型
- 添加类型信息的回填机制

---

### 方案B：增强类型推断逻辑（中优先级）

**目标**：在类型检查阶段，对 `CN_TYPE_UNKNOWN` 类型进行推断

**修改文件**：

#### 1. `src/semantics/symbols/type_system.c` - 类型系统

**修改内容**：
- 在 `cn_type_compatible()` 函数中添加对 `CN_TYPE_UNKNOWN` 作为实际参数类型的处理

**修改代码**（第275行附近）：

```c
// UNKNOWN 类型可以接受任何类型（用于内置函数如 类型大小）
if (b->kind == CN_TYPE_UNKNOWN) {
    return true;
}

// 【新增】UNKNOWN 类型作为实际参数时，尝试推断类型
// 当实际参数类型为 UNKNOWN 时，如果形式参数类型已知，允许通过
// 这通常发生在模块导入时类型信息未完整解析的情况
if (a->kind == CN_TYPE_UNKNOWN) {
    // 记录警告信息，但不阻止编译
    // TODO: 后续可以添加类型推断逻辑
    return true;  // 临时允许，让编译继续
}
```

**风险**：
- 可能掩盖真正的类型错误
- 需要配合警告信息使用

---

### 方案C：放宽类型兼容性检查（低优先级）

**目标**：在特定场景下允许 `CN_TYPE_UNKNOWN` 类型

**修改文件**：

#### 1. `src/semantics/checker/semantic_passes.c` - 语义检查

**修改内容**：
- 在函数调用参数检查时，对 `CN_TYPE_UNKNOWN` 类型添加特殊处理
- 发出警告而非错误

**修改位置**：函数调用参数类型检查逻辑（约1594-1652行）

## 四、推荐修复顺序

### 阶段一：快速修复（方案B）

2. **调查模块导入的类型解析流程**

3. **修复模块加载器**
   - 确保导出符号的类型信息完整

4. **修复作用域构建器**

   - 帮助开发者定位问题
