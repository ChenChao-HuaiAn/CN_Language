## 1. 概述

### 1.1 需求来源

根据 [`005 CN语言静态关键字必要性分析.md`](plans/005 CN语言静态关键字必要性分析.md) 的分析结论：

| `静态` 关键字用途 | 建议 | 原因 |
|-------------------|------|------|
| 文件级可见性控制 | ❌ 不实现 | 已被模块系统 `私有:` 块完全覆盖 |
| 函数内静态变量 | ✅ 建议实现 | 模块系统无法替代，有独特价值 |

### 1.2 实现范围

**实现功能**：
- 函数内部的静态局部变量声明
- 变量在首次调用时初始化
- 后续调用保持状态

**不实现功能**：
- 文件级静态变量/函数（已被模块系统覆盖）
- 类/结构体静态成员（暂不支持）

### 1.3 语法设计

```cn
函数 计数器() -> 整数 {
    静态 整数 计数 = 0;  // 只初始化一次
    计数 = 计数 + 1;
    返回 计数;
}

// 调用结果：1, 2, 3, 4...
```

**语法规则**：
```
静态变量声明 ::= "静态" 类型? 标识符 ("=" 表达式)? ";"
```

### 4.3 步骤3：语法分析器修改

#### 4.3.1 修改 parse_statement 函数

文件：[`src/frontend/parser/parser.c`](src/frontend/parser/parser.c:1061)

**修改位置**：在变量声明解析的开头添加 `静态` 关键字处理

```c
// 在 parse_statement 函数中，修改变量声明解析条件
if (parser->current.kind == CN_TOKEN_KEYWORD_STATIC ||
    parser->current.kind == CN_TOKEN_KEYWORD_CONST ||
    parser->current.kind == CN_TOKEN_KEYWORD_VAR ||
    parser->current.kind == CN_TOKEN_KEYWORD_INT ||
    parser->current.kind == CN_TOKEN_KEYWORD_FLOAT ||
    parser->current.kind == CN_TOKEN_KEYWORD_STRING ||
    parser->current.kind == CN_TOKEN_KEYWORD_BOOL) {
    
    const char *var_name;
    size_t var_name_length;
    CnAstExpr *initializer = NULL;
    CnType *declared_type = NULL;
    int is_const = 0;
    int is_static = 0;  // 【新增】静态标记

    // 【新增】处理 '静态' 关键字
    if (parser->current.kind == CN_TOKEN_KEYWORD_STATIC) {
        is_static = 1;
        parser_advance(parser);
        
        // 静态变量后可以跟类型关键字或 '变量' 关键字
        // 支持：静态 整数 x = 0; 或 静态 变量 x = 0;
        if (parser->current.kind == CN_TOKEN_KEYWORD_VAR) {
            parser_advance(parser);
            declared_type = NULL;  // 后续通过类型推断
        } else if (parser->current.kind == CN_TOKEN_KEYWORD_INT ||
                   parser->current.kind == CN_TOKEN_KEYWORD_FLOAT ||
                   parser->current.kind == CN_TOKEN_KEYWORD_STRING ||
                   parser->current.kind == CN_TOKEN_KEYWORD_BOOL) {
            declared_type = parse_type(parser);
            if (!declared_type) {
                return NULL;
            }
        } else {
            // 静态关键字后必须有类型或变量关键字
            parser->error_count++;
            if (parser->diagnostics) {
                cn_support_diagnostics_report(parser->diagnostics,
                    CN_DIAG_SEVERITY_ERROR,
                    CN_DIAG_CODE_PARSE_INVALID_VAR_DECL,
                    parser->lexer ? parser->lexer->filename : NULL,
                    parser->current.line,
                    parser->current.column,
                    "语法错误：静态关键字后必须有类型或变量关键字");
            }
            return NULL;
        }
    }
    // 处理 '常量' 关键字（现有代码）
    else if (parser->current.kind == CN_TOKEN_KEYWORD_CONST) {
        is_const = 1;
        parser_advance(parser);
        // ... 现有常量处理逻辑 ...
    }
    // ... 后续现有代码 ...
    
    // 创建变量声明节点时，设置 is_static 字段
    CnAstStmt *stmt = make_var_decl_stmt(var_name, var_name_length, 
                                          declared_type, initializer);
    if (stmt) {
        stmt->as.var_decl.is_const = is_const;
        stmt->as.var_decl.is_static = is_static;  // 【新增】
    }
    return stmt;
}
```

```c
    // 生成函数签名
    const char *ret_type = get_c_type_string(func->return_type);
    fprintf(ctx->output_file, "%s cn_func_%s(", ret_type, func->name);
    
    // 生成参数列表
    // ...
    
    fprintf(ctx->output_file, ") {\n");
    
    // 【新增】生成静态变量声明（在函数体开头）
    if (func->static_var_count > 0) {
        fprintf(ctx->output_file, "    /* 静态局部变量 */\n");
        for (size_t i = 0; i < func->static_var_count; i++) {
            CnIrOperand *var = &func->static_vars[i];
            const char *type_str = get_c_type_string(var->type);
            fprintf(ctx->output_file, "    static %s cn_static_%s_%s",
                    type_str, func->name, var->name);
            
            // 如果有初始化值，生成初始化
            if (var->has_initializer) {
                fprintf(ctx->output_file, " = ");
                cn_cgen_operand(ctx, var->initializer);
            }
            fprintf(ctx->output_file, ";\n");
        }
        fprintf(ctx->output_file, "\n");
    }
    
    // 生成局部变量声明
    // ...
    
    // 生成函数体
    // ...
    
    fprintf(ctx->output_file, "}\n");
}
```

#### 4.6.2 静态变量命名规则

为避免命名冲突，静态变量使用以下命名规则：

```c
// 命名格式：cn_static_{函数名}_{变量名}
// 例如：函数 计数器() 中的静态变量 计数
// 生成：static int cn_static_计数器_计数 = 0;
```

## 附录A：C语言静态变量语义参考

### A.1 C语言静态变量特性

```c
void example() {
    static int count = 0;  // 只初始化一次
    count++;               // 值在调用间保持
}

// 等价于：
static int __static_example_count = 0;  // 全局存储
void example() {
    __static_example_count++;
}
```

### A.2 初始化时机

- **首次调用初始化**：C++11 后保证线程安全的首次调用初始化
- **编译时初始化**：如果初始化值是常量表达式，在编译时完成
- **零初始化**：如果没有显式初始化，自动零初始化

## 附录B：与常量关键字对比

| 特性 | 常量 | 静态 |
|------|------|------|
| 必须初始化 | ✅ 是 | ❌ 否（默认零值） |
| 初始化表达式 | 必须是编译时常量 | 必须是编译时常量 |
| 可修改 | ❌ 否 | ✅ 是 |
| 作用域 | 任意位置 | 仅函数内部 |
| 生命周期 | 块作用域 | 程序运行期 |
| 存储位置 | 栈或常量区 | 静态存储区 |

## 附录C：实现注意事项

### C.1 线程安全考虑

当前实现不保证线程安全的首次初始化。如果未来需要支持多线程，可以考虑：

1. 使用编译器提供的线程局部存储（TLS）
2. 生成线程安全的初始化代码（类似 C++11 的 magic statics）

### C.2 调试支持

静态变量命名应包含函数名，便于调试：

```c
// CN 代码
函数 计数器() {
    静态 整数 计数 = 0;
}

// 生成的 C 代码
static int cn_static_计数器_计数 = 0;
```

### C.3 与模块系统的交互

静态变量与模块私有变量的区别：

```cn
// 模块私有变量 - 可被模块内其他函数访问
模块 示例 {
私有:
    整数 模块计数 = 0;
    
    函数 递增() {
        模块计数 = 模块计数 + 1;
    }
}

// 静态局部变量 - 只能在声明它的函数内访问
函数 计数器() {
    静态 整数 局部计数 = 0;  // 外部无法访问
    局部计数 = 局部计数 + 1;
}
```
