build\src\Release\cnc.exe --project src_cn --main src_cn\主程序\cnc.cn -I src_cn -o build/bootstrap/cn_v1.exe --emit-c

## 错误分类

### 1. 未声明的标识符 (约30个)

**典型错误**:
```
error C2065: "cn_var_布尔字面量": 未声明的标识符
error C2065: "cn_var_关键字_模块": 未声明的标识符
error C2065: "cn_var_私有可见": 未声明的标识符
error C2065: "cn_var_公开可见": 未声明的标识符
error C2065: "cn_var_默认可见": 未声明的标识符
```

**根本原因**:
- CN源代码中使用枚举成员（如 `布尔字面量`、`关键字_模块`）作为值
- 编译器生成的C代码将这些枚举成员引用转换为 `cn_var_枚举成员名` 变量
- 但这些变量从未被定义

**示例**:
```cn
// CN源代码 (表达式解析.cn:620)
节点.字面量表达式.类型 = 布尔字面量;

// 生成的C代码 (表达式解析.c:2954)
r14 = cn_var_布尔字面量;  // 错误：cn_var_布尔字面量 未定义
```

### 2. 枚举类型重定义 (约5个)

**典型错误**:
```
error C2011: "可见性":"enum"类型重定义
```

**根本原因**:
- `enum 可见性` 在多个模块中被定义
- 每个生成的.c文件都包含完整的枚举定义
- 当多个.c文件一起编译时，枚举被重复定义

### 3. 结构体成员访问错误 (约100个)

**典型错误**:
```
error C2039: "节点基类": 不是 "声明节点" 的成员
error C2039: "表达式": 不是 "语句节点" 的成员
error C2039: "函数名称": 不是 "声明节点" 的成员
error C2224: ".类型"的左侧必须具有结构/联合类型
error C2223: "->名称"的左侧必须指向结构/联合
```

**根本原因**:
- 生成的C代码中结构体定义不完整或成员访问方式错误
- 联合体(union)成员访问使用了错误的语法

### 4. 类型转换错误 (约50个)

**典型错误**:
```
error C2440: "=": 无法从"表达式节点 *"转换为"参数"
error C2440: "函数": 无法从"__int64"转换为"源位置"
error C2440: "函数": 无法从"参数节点 *"转换为"参数"
```

**根本原因**:
- 生成的C代码中类型不匹配
- 指针类型和结构体类型混淆

### 5. 语法错误 (约20个)

**典型错误**:
```
error C2059: 语法错误:";"
```

**根本原因**:
- 生成的C代码中存在空语句或缺失表达式
- 例如：`r16 = r6 == /* NONE */;` (声明解析.c:1041)

### 6. 函数参数不匹配 (约10个)

**典型错误**:
```
error C2198: "报告错误": 用于调用的参数太少
error C2198: "cn_rt_malloc": 用于调用的参数太少
```

**根本原因**:
- 函数调用时参数数量或类型不正确

## 问题根源分析

### 核心问题：枚举成员引用处理错误

CN语言中，枚举成员可以直接作为值使用：
```cn
枚举 字面量类型 {
    字面量_整数,
    字面量_浮点,
    字面量_布尔,  // 这个成员应该被识别为枚举值
    ...
}

// 使用枚举成员
节点.字面量表达式.类型 = 字面量_布尔;  // 字面量_布尔 应该是枚举值
```

但编译器生成的C代码错误地将其处理为变量引用：
```c
// 错误的生成结果
r14 = cn_var_布尔字面量;  // 应该是 字面量类型_字面量_布尔
```

正确的生成应该是：
```c
// 正确的生成结果
r14 = 字面量类型_字面量_布尔;  // 使用枚举成员的正确名称
```

### 次要问题：跨模块类型定义

每个模块生成的.c文件都包含完整的类型定义，导致链接时类型重定义错误。

## 修复建议

### 方案A：修复枚举成员引用生成

修改代码生成器，正确处理枚举成员引用：

1. 识别枚举类型上下文
2. 生成正确的枚举成员名称（`枚举类型名_成员名`）
3. 确保枚举定义在头文件中，避免重定义

### 方案B：添加枚举成员变量定义

在生成的C代码中，为每个枚举成员定义对应的常量变量：

```c
// 在枚举定义后添加
static const enum 字面量类型 cn_var_布尔字面量 = 字面量类型_字面量_布尔;
```

### 方案C：使用头文件共享类型定义

1. 将公共类型定义提取到共享头文件
2. 每个.c文件只包含头文件引用
3. 避免类型重定义

## 下一步行动

1. **优先级高**：修复枚举成员引用生成逻辑
2. **优先级中**：解决类型重定义问题
3. **优先级低**：修复结构体成员访问和类型转换错误

## 结论

阶段1编译失败，主要原因是编译器在处理枚举成员引用时存在bug。需要修复编译器的代码生成逻辑，才能继续自举测试。
